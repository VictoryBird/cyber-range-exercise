### 2-1. 네트워크 설정 (고정 IP)

**운영자 PC 1 (Windows 10):**


**운영자 PC 2 (Windows 11):**

### 2-2. 브라우저 자동 실행 설정

시스템 시작 시 SCADA HMI 화면을 자동으로 표시하도록 설정한다.

**시작프로그램 배치 파일 (startup_hmi.bat):**

```batch
@echo off
REM OT 운영자 PC - SCADA HMI 자동 실행
REM 배치 경로: C:\Users\operator\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\startup_hmi.bat

REM SCADA 서버 접속 대기 (부팅 시 네트워크 안정화)
timeout /t 30 /nobreak

REM SCADA-LTS HMI 전체 화면으로 실행
start "" "C:\Program Files\Google\Chrome\Application\chrome.exe" ^
    --kiosk ^
    --no-first-run ^
    --disable-session-crashed-bubble ^
    --disable-infobars ^
    "http://192.168.92.247:8080/ScadaLTS/"

REM Node-RED 대시보드 (보조 모니터용, 선택)
REM start "" "C:\Program Files\Google\Chrome\Application\chrome.exe" ^
REM     "http://192.168.92.247:1880/ui"
```

## 4. 운영자 관점의 공격 영향

운영자 PC 자체에는 직접적인 취약점이 없으나, SCADA/PLC 데이터가 조작되면 운영자는 잘못된 정보에 기반하여 판단하게 된다.

```
정상 상태:
+--------------------------------------------------+
| 운영자 화면     | 실제 상태     | 운영자 판단     |
|-----------------|---------------|-----------------|
| 온도: 25.3 C    | 온도: 25.3 C  | 정상 -> 유지    |
| 압력: 100 bar   | 압력: 100 bar | 정상 -> 유지    |
+--------------------------------------------------+

공격 상태 (데이터 조작 후):
+--------------------------------------------------+
| 운영자 화면     | 실제 상태     | 운영자 판단     |
|-----------------|---------------|-----------------|
| 온도: 25.0 C    | 온도: 85.0 C  | 정상 -> 유지    |
|  (주입된 값)    |  (위험!)      |  (잘못된 판단)  |
| 압력: 100 bar   | 압력: 150 bar | 정상 -> 유지    |
|  (주입된 값)    |  (위험!)      |  (잘못된 판단)  |
+--------------------------------------------------+
  -> 운영자는 정상으로 인지하지만 실제 공정은 위험 상태
  -> 대응 지연으로 사고 발생 가능
```

## 7. 블루팀 탐지 포인트 (운영자 PC 관련)

운영자 PC 자체가 공격 대상은 아니지만, SCADA/PLC 측 공격으로 인해 운영자가 잘못된 정보를 수신하게 된다. 블루팀은 다음을 감시해야 한다.

### 7-1. 간접 탐지 (SCADA/PLC 로그 교차 확인)

운영자 PC에서의 비정상 행동은 SCADA 서버 로그에서 탐지한다.

| 탐지 항목 | 확인 위치 | 설명 |
|-----------|-----------|------|
| 비인가 관리자 로그인 | SCADA Tomcat 액세스 로그 | 운영자 계정(operator1/2)이 아닌 admin 로그인 시도 |
| 비정상 API 호출 | SCADA Tomcat 액세스 로그 | 운영자 PC IP에서 PUT/POST API 호출 (읽기 전용 계정인데 쓰기 시도) |
| Node-RED 접근 | Node-RED 서비스 로그 | 운영자 PC에서 :1880 편집기 접근 (대시보드가 아닌 편집기) |

### 7-2. 탐지 시나리오 매핑

```
공격으로 인한 운영자 영향:
+-----------------------+     +-----------------------------------+
| 공격 행위             | --> | 운영자에 미치는 영향              |
+-----------------------+     +-----------------------------------+
| PLC 센서 범위 조작    | --> | HMI에 잘못된 값 표시              |
| PLC 값 주입           | --> | HMI에 고정된 정상값 표시          |
| SCADA 태그 값 조작    | --> | HMI에 조작된 값 직접 표시         |
| SCADA 알람 임계값 변경| --> | 위험 상태인데 알람 미발생         |
| Node-RED 플로우 변경  | --> | 데이터 전달 경로 변조/중단        |
+-----------------------+     +-----------------------------------+

블루팀 대응:
  -> 운영자 PC 화면의 값과 PLC 직접 조회값 비교
  -> SCADA 태그 이력에서 급격한 변화/값 고정 패턴 확인
  -> 알람 임계값이 원래 설정값과 다른지 확인
```

### 7-3. 블루팀 대응 체크리스트 (운영자 PC 관련)

| 순서 | 탐지 항목 | 확인 방법 | 대응 조치 |
|------|-----------|-----------|-----------|
| 1 | HMI 표시값 이상 | 운영자 수동 확인 / PLC 직접 조회 비교 | SCADA 데이터 포인트 검증 |
| 2 | 알람 미발생 의심 | 알람 임계값 설정 확인 | 원래 임계값으로 복원 |
| 3 | 운영자 PC에서 비인가 활동 | SCADA Tomcat 로그에서 운영자 IP 필터 | 운영자 계정 비밀번호 변경, 접근 제한 |
