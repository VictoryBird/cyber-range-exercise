## 1. 개요

### 1.1 자산 목적

D4(Cowrie SSH 허니팟)와 D5(SNARE/Tanner Web 허니팟)는 공격자의 행동을 기록하고 분석하기 위한 **디셉션(Deception) 자산**이다. 이 자산들은 공격 대상이 아니라, 공격자가 군 DMZ 내에서 탐색(정찰) 활동을 할 때 자연스럽게 접근하도록 유도하는 함정이다.

### 1.2 훈련에서의 역할

- **블루팀 훈련**: 허니팟 로그 분석을 통한 공격자 TTP(전술/기법/절차) 파악 능력 향상
- **공격 탐지**: 허니팟 접근 자체가 공격 징후 — 정상 사용자는 접근할 이유가 없음
- **IOC 수집**: 공격자가 사용하는 도구, 명령어, 파일 등을 수집

### 1.3 네트워크 위치

```

    │
    ├── D1 외부 포털 (211.57.64.10)    ← 실제 서비스
    ├── D2 VPN 게이트웨이 (211.57.64.11) ← 실제 서비스
    ├── D3 자료교환 (211.57.64.12)      ← 실제 서비스
    ├── D4 허니팟 SSH (211.57.64.13)    ← 허니팟 (Cowrie)
    └── D5 허니팟 Web (211.57.64.14)    ← 허니팟 (SNARE/Tanner)
```

공격자가 nmap으로 211.57.64.0/24 대역을 스캔하면 D4/D5도 발견된다. SSH(22)와 HTTP(80/443)가 응답하므로 공격자는 실제 서비스로 오인하고 공격을 시도할 수 있다.

## 2. D4: Cowrie SSH 허니팟 (211.57.64.13)

### 2.1 기술 스택

| 구성요소 | 버전/사양 | 용도 |
|----------|-----------|------|
| Ubuntu | 22.04 LTS | 호스트 운영체제 |
| Docker | 24.0.x | 컨테이너 런타임 |
| Cowrie | 2.5.x | SSH/Telnet 허니팟 |
| Python | 3.11.x | Cowrie 런타임 |

### 2.2 Cowrie 구성

Cowrie는 중간 상호작용(medium-interaction) SSH 허니팟으로, 공격자에게 가짜 SSH 셸을 제공하고 모든 입력/출력을 기록한다.

**핵심 설계 원칙:**
- 약한 자격증명을 설정하여 공격자의 접근을 유도
- 가짜 파일 시스템을 제공하여 공격자가 실제 서버라고 착각하도록 함
- 모든 세션을 녹화하여 블루팀 분석에 활용

### 2.3 Docker Compose 설정

```yaml
# docker-compose.yml (D4: 211.57.64.13)
version: '3.8'

services:
  cowrie:
    image: cowrie/cowrie:latest
    container_name: cowrie-ssh
    restart: unless-stopped
    ports:
      - "22:2222"          # 외부 22 → 내부 2222
      - "23:2223"          # Telnet (선택)
    volumes:
      - ./cowrie/etc:/cowrie/cowrie-git/etc
      - ./cowrie/var/log:/cowrie/cowrie-git/var/log/cowrie
      - ./cowrie/var/lib/cowrie/downloads:/cowrie/cowrie-git/var/lib/cowrie/downloads
      - ./cowrie/var/lib/cowrie/tty:/cowrie/cowrie-git/var/lib/cowrie/tty
      - ./cowrie/honeyfs:/cowrie/cowrie-git/honeyfs
    environment:
      - COWRIE_TELNET_ENABLED=no
    networks:
      - honeypot

networks:
  honeypot:
    driver: bridge
```

### 2.4 Cowrie 주요 설정 (`cowrie.cfg`)

```ini
# cowrie/etc/cowrie.cfg

[honeypot]
# 호스트명 (실제 군 서버처럼 위장)
hostname = mil-mgmt-svr
log_path = var/log/cowrie
download_path = var/lib/cowrie/downloads
ttylog = true
ttylog_path = var/lib/cowrie/tty
interactive_timeout = 300
authentication_timeout = 120

# 허니팟 유형: shell (가짜 셸 제공)
backend = shell

[shell]
# 가짜 파일 시스템
filesystem = share/cowrie/fs.pickle

[ssh]
# SSH 설정
enabled = true
listen_endpoints = tcp:2222:interface=0.0.0.0
version = SSH-2.0-OpenSSH_8.9p1 Ubuntu-3ubuntu0.6
sftp_enabled = true

[telnet]
enabled = false

[output_jsonlog]
enabled = true
logfile = var/log/cowrie/cowrie.json

[output_textlog]
enabled = true
logfile = var/log/cowrie/cowrie.log
```

### 2.6 가짜 파일 시스템

Cowrie는 `fs.pickle` 파일로 가짜 파일 시스템을 시뮬레이션한다. 군 서버처럼 보이도록 다음 파일/디렉터리를 배치한다:

```
/
├── etc/
│   ├── hostname          → "mil-mgmt-svr"
│   ├── hosts             → 군 내부 서버 IP 매핑 (페이크)
│   ├── passwd            → 가짜 사용자 목록
│   └── shadow            → 가짜 해시
├── home/
│   ├── admin/
│   │   ├── .bash_history → 가짜 명령어 이력
│   │   └── notes.txt     → "VPN 접속 정보: vpn.mnd.valdoria.mil"
│   └── operator/
│       └── scripts/
│           └── backup.sh → 가짜 백업 스크립트
├── var/
│   └── log/
│       ├── auth.log      → 가짜 인증 로그
│       └── syslog        → 가짜 시스템 로그
├── opt/
│   └── mil-tools/
│       └── README.txt    → "군 관리 도구 v2.1"
└── tmp/
```

```yaml

      - ./snare/logs:/opt/snare/logs
```

```bash
  renderer: networkd

        - 211.57.64.13/24
      routes:
        - to: default
          via: 211.57.64.1
      nameservers:
        addresses:
          - 192.168.1.20

curl -fsSL https://get.docker.com | sh
```
