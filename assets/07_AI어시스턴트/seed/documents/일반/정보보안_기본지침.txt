제1장 총칙

제1조 (목적)
본 지침은 행정안전부의 정보자산을 보호하고 사이버 위협에 대응하기 위한
기본 보안 규정을 수립하는 것을 목적으로 한다.

제2조 (적용 범위)
본 지침은 행정안전부 소속 전 직원, 파견 직원, 외부 용역 인력 및
정보시스템에 접근하는 모든 사용자에게 적용된다.

----------------------------------------------------------------

제2장 계정 관리

제3조 (계정 발급)
1. 모든 직원은 입사 시 고유 계정을 발급받는다.
2. 계정 형식: 성(영문) + 이름(영문 이니셜) + 사번 끝 4자리
   예) 홍길동(사번 20210001) → honggd0001
3. 공용 계정 사용은 원칙적으로 금지한다.

제4조 (비밀번호 정책)
1. 최소 길이: 12자 이상
2. 구성 요건: 영문 대/소문자, 숫자, 특수문자 중 3종 이상 조합
3. 변경 주기: 90일
4. 이전 5회 비밀번호 재사용 금지
5. 5회 연속 인증 실패 시 계정 잠금 (30분)
6. 초기 비밀번호 발급 후 최초 로그인 시 변경 필수

제5조 (계정 관리)
1. 장기 미사용 계정(90일 이상)은 비활성화
2. 퇴직/전출 시 즉시 계정 삭제
3. 계정 정보(ID, 비밀번호) 타인 공유 절대 금지
4. 이상 접속 탐지 시 보안정책과에 즉시 신고

----------------------------------------------------------------

제3장 네트워크 보안

제6조 (네트워크 접근 통제)
1. 내부망은 인가된 단말기만 접속 허용
2. MAC 주소 인증 적용 (NAC)
3. 무선 네트워크 사용은 승인 후에만 허용

제7조 (VPN 사용)
1. 외부에서 내부망 접속 시 VPN 사용 필수
2. VPN 계정은 별도 신청 절차를 통해 발급
3. VPN 접속 로그는 6개월간 보관
4. VPN 접속 시 MFA(다중 인증) 적용

제8조 (인터넷 사용)
1. 내부망과 인터넷망은 물리적으로 분리
2. 인터넷 사용은 별도 인터넷 PC에서만 가능
3. 파일 전송은 망간 자료전송 시스템을 통해서만 허용

----------------------------------------------------------------

제4장 단말기 보안

제9조 (PC 보안)
1. 운영체제 및 백신 소프트웨어 자동 업데이트 설정
2. 화면보호기 설정: 5분 이상 미사용 시 자동 잠금
3. USB 등 이동식 저장매체 사용 통제
4. 개인 소유 장비(BYOD)의 내부망 접속 금지

제10조 (모바일 기기)
1. 업무용 모바일 기기는 MDM(Mobile Device Management) 설치 필수
2. 비인가 앱 설치 금지
3. 분실/도난 시 즉시 원격 삭제(Remote Wipe) 요청

----------------------------------------------------------------

제5장 정보 분류 및 보호

제11조 (정보 분류 등급)
1. 공개: 외부 공개 가능 정보
2. 내부용: 부 직원만 접근 가능
3. 대외비: 부 외부 유출 금지
4. 비밀: 법령에 의한 비밀 정보 (1급/2급/3급)

제12조 (문서 보호)
1. 대외비 이상 문서는 암호화하여 저장
2. 문서 출력 시 워터마크 적용
3. 문서 외부 전송 시 승인 절차 필수
4. Nextcloud 공유 설정 시 최소 권한 원칙 적용

제13조 (데이터 백업)
1. 주요 데이터는 주 1회 이상 백업
2. 백업 데이터는 별도 보안 구역에 보관
3. 분기 1회 복원 테스트 수행

----------------------------------------------------------------

제6장 사고 대응

제14조 (보안 사고 분류)
1. 경미: 바이러스 감염, 스팸 메일 등
2. 중대: 정보 유출, 시스템 해킹, 서비스 장애 등
3. 심각: 대규모 정보 유출, 랜섬웨어 감염, APT 공격 등

제15조 (사고 보고 절차)
1. 발견 즉시 보안정책과(내선 3100)에 신고
2. 1시간 이내 초기 보고서 작성
3. 24시간 이내 상세 분석 보고서 제출
4. 보안 사고 은닉 시 가중 징계

제16조 (사고 대응 체계)
1. 보안정책과가 사고 대응 총괄
2. 필요시 유관기관(KISA, 경찰 등) 협조 요청
3. 사고 종결 후 30일 이내 재발 방지 대책 수립
