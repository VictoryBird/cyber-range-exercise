## 1. 개요

### 1.1 자산 목적

D1 외부 포털 서버는 가상 국가 Valdoria의 국방부(MND) 대표 홈페이지를 호스팅하는 웹 서버이다. 전자정부 표준프레임워크(eGovframework)를 기반으로 구축되며, 한국 국방부 공식 홈페이지와 유사한 구조를 제공한다.

주요 기능:
- **공지사항 게시판**: 국방 정책, 보도자료, 입찰 공고
- **연락처 안내**: 부서별 연락처, 오시는 길
- **자료실**: 공개 자료 다운로드 (양식, 규정집 등)
- **민원 조회**: 민원 접수번호 기반 처리 상태 확인

### 1.2 훈련에서의 역할

이 자산은 PHASE 4의 **배경 취약점**으로 설계되었다. Spring4Shell(CVE-2022-22965) 취약점을 의도적으로 탑재하여, 공격자가 군 DMZ에 대한 추가 공격 벡터를 확보할 수 있도록 한다. 다만 VPN 게이트웨이(D2)가 주요 피벗 포인트이므로, D1은 보조적 공격 경로로 기능한다.

## 4. Spring4Shell 취약점 설계 (CVE-2022-22965)

### 4.1 취약점 개요

| 항목 | 내용 |
|------|------|
| CVE | CVE-2022-22965 |
| 별칭 | Spring4Shell, SpringShell |
| 영향 버전 | Spring Framework 5.3.0 ~ 5.3.17, 5.2.0 ~ 5.2.19 |
| 적용 버전 | **Spring Framework 5.3.17** (본 자산) |
| CVSS 3.1 | 9.8 (Critical) |
| 공격 전제 조건 | JDK 9+, WAR 배포 (내장 서버 아님), Tomcat 서블릿 컨테이너 |
| 본 자산 충족 여부 | **JDK 11 + WAR on Tomcat 9 = 모든 조건 충족** |

### 4.2 취약점 원리

Spring4Shell은 Spring Framework의 데이터 바인딩 메커니즘을 악용한다. Spring MVC에서 HTTP 요청 파라미터를 Java 객체에 자동 바인딩할 때, `class.module.classLoader` 체인을 통해 Tomcat의 `AccessLogValve` 속성을 조작할 수 있다. 이를 통해 임의 위치에 JSP 웹셸 파일을 생성한다.

**공격 체인:**

```
HTTP POST 요청
    │
    ▼
Spring DataBinder (파라미터 바인딩)
    │
    ▼
class.module.classLoader.resources.context.parent.pipeline.first.*
    │  (Tomcat AccessLogValve 속성 접근)
    │
    ├─ .pattern = %{c2}i (웹셸 코드 — Request Header에서 주입)
    ├─ .suffix = .jsp
    ├─ .directory = webapps/ROOT
    ├─ .prefix = shell
    ├─ .fileDateFormat = (빈 문자열)
    │
    ▼
Tomcat AccessLogValve가 shell.jsp 파일 생성
    │
    ▼
공격자가 shell.jsp 접근 → RCE 달성
```

### 4.3 취약 조건 구성

본 자산이 Spring4Shell에 취약하려면 다음 조건이 모두 충족되어야 한다:

1. **JDK 9 이상**: JDK 9에서 도입된 `Module` 클래스가 `class.module` 경로를 노출
   - 본 자산: OpenJDK **11** 사용 → **충족**

2. **WAR 배포**: 내장 서버(embedded)가 아닌 외부 Tomcat에 WAR로 배포
   - 본 자산: eGov WAR를 Tomcat 9에 배포 → **충족**

3. **Spring Framework 5.3.17 이하**:
   - 본 자산: **5.3.17** 정확히 사용 → **충족**

4. **데이터 바인딩 사용 컨트롤러**: `@ModelAttribute` 또는 POJO 파라미터 바인딩
   - 본 자산: eGov 검색 컨트롤러에서 `SearchVO` 객체 바인딩 → **충족**
