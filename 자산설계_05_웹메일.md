## 1. 개요

### 1.1 자산 목적

웹메일 서버는 발도리아(Valdoria) 행정안전부(MOIS) 내부 직원을 위한 이메일 서비스를 제공하는 서버이다. Postfix(MTA), Dovecot(IMAP), Roundcube(WebUI)의 3개 오픈소스 구성 요소를 조합하여 메일 송수신, 저장 및 웹 기반 열람 기능을 제공한다.

주요 기능:

- **메일 송수신**: Postfix SMTP를 통한 내부 메일 라우팅 및 배달
- **메일 저장/열람**: Dovecot IMAP을 통한 메일함 접근
- **웹 인터페이스**: Roundcube를 통한 브라우저 기반 메일 읽기/쓰기/관리
- **AD 인증 연동**: Active Directory(192.168.100.50)를 통한 통합 계정 인증

### 1.2 훈련에서의 역할

이 자산은 레드팀의 **내부 피싱 메일 발송 거점(Phishing Relay Point)**으로 설계되었다. 공격자는 내부 업무 포털(192.168.100.10)에서 탈취한 계정 크리덴셜을 재사용하여 웹메일에 로그인하고, 업무용 PC(192.168.100.31~35)를 대상으로 악성 첨부파일이 포함된 피싱 메일을 발송한다.

블루팀 관점에서는 Postfix/Dovecot 인증 로그와 Roundcube 접근 로그를 분석하여 비정상적 로그인 시간, 대량 메일 발송, 동시 세션 등의 이상 징후를 탐지해야 하는 핵심 분석 대상이다.

### 3.2 메일 수신 흐름

```
1. 외부/내부 발신자 → Postfix (:25) SMTP 수신
2. Postfix → smtpd_recipient_restrictions 검사 (내부 도메인만 수신)
3. Postfix → virtual_transport = lmtp:unix:private/dovecot-lmtp
4. Dovecot LMTP → Maildir (/var/mail/vhosts/mois.local/{user}/)
5. 사용자 → Roundcube (HTTP:80) 또는 IMAP 클라이언트 → Dovecot (:143/:993)
6. Dovecot → LDAP 인증 (192.168.100.50:389) → 인증 성공 시 메일함 반환
```

### 3.3 메일 발신 흐름 (Roundcube → 내부 수신자)

```
1. 사용자가 Roundcube에서 메일 작성 → "보내기" 클릭
2. Roundcube → Postfix (:25) SMTP 제출 (smtp_server = localhost)
3. Postfix → 수신자 도메인 확인 (mois.local = 로컬 배달)
4. Postfix → Dovecot LMTP → 수신자 Maildir에 저장
```

### 3.4 AD 인증 흐름

```
1. 사용자가 Roundcube 로그인 폼에 ID/PW 입력
2. Roundcube → Dovecot IMAP 로그인 시도 (PLAIN 인증)
3. Dovecot → passdb ldap 조회 (192.168.100.50:389)
4. LDAP 바인드: uid={user},ou=Users,dc=mois,dc=local + 입력된 비밀번호
5. 바인드 성공 → Dovecot 인증 성공 → Roundcube 세션 생성
6. 바인드 실패 → 인증 실패 → 로그 기록
```

## 6. 메일 서비스 설정 명세

### 6.1 Postfix 설정

#### `/etc/postfix/main.cf`

```ini
# ============================================================
# Postfix main.cf — 웹메일 서버 (192.168.100.11)
# 훈련 환경용 설정 (의도적 취약점 포함)
# ============================================================

# 기본 설정
smtpd_banner = $myhostname ESMTP $mail_name (Debian/GNU)
biff = no
append_dot_mydomain = no
readme_directory = no

# 호스트/도메인 설정
myhostname = mail.mois.local
mydomain = mois.local
myorigin = $mydomain
mydestination = localhost
mynetworks = 127.0.0.0/8, 192.168.100.0/24

# [취약] mynetworks에 내부 서브넷 전체 포함
#        → 내부 IP를 가진 어떤 호스트든 인증 없이 메일 발송 가능

# TLS 설정
smtpd_tls_cert_file = /etc/ssl/certs/mail.crt
smtpd_tls_key_file = /etc/ssl/private/mail.key
smtpd_tls_security_level = may
smtp_tls_security_level = may

# SASL 인증 설정 (Dovecot 연동)
smtpd_sasl_type = dovecot
smtpd_sasl_path = private/auth
```

## 8. 의도적 취약점 설계

### 8.1 SMTP 발송 속도 제한 없음 (대량 피싱 가능)

**취약 구성 위치**: `/etc/postfix/main.cf`

```ini
# 기본값 유지 = 제한 없음
# smtpd_client_message_rate_limit 미설정 (기본: 0 = 무제한)
# smtpd_client_recipient_rate_limit 미설정 (기본: 0 = 무제한)
# anvil_rate_time_unit = 60s
```

**공격 활용**: 레드팀이 탈취 계정으로 Roundcube에 로그인한 후, 전 직원에게 피싱 메일을 5분 이내에 일괄 발송할 수 있다. 정상 환경에서는 분당 발송 수를 제한하여 이를 방지해야 한다.

**정상 설정 예시** (훈련 후 개선 지침):
```ini
smtpd_client_message_rate_limit = 10
smtpd_client_recipient_rate_limit = 20
```

---

### 8.2 내부→내부 SMTP 릴레이 추가 인증 없음

**취약 구성 위치**: `/etc/postfix/main.cf`

```ini
mynetworks = 127.0.0.0/8, 192.168.100.0/24

smtpd_recipient_restrictions =
    permit_mynetworks,          # ← 내부 IP는 무조건 허용
    permit_sasl_authenticated,
    reject_unauth_destination
```

**공격 활용**: `192.168.100.0/24` 대역의 어떤 호스트든 SMTP AUTH 없이 `mail.mois.local:25`로 메일을 발송할 수 있다. 민원 처리 서버(192.168.100.x)를 장악한 후 직접 SMTP 명령으로 피싱 메일을 발송하는 것도 가능하다.

```bash
# 공격 예시: 장악한 내부 서버에서 직접 SMTP로 피싱 메일 발송
telnet 192.168.100.11 25
EHLO attacker
MAIL FROM:<jung.dongwook@mois.local>
RCPT TO:<hong.gildong@mois.local>
DATA
Subject: [긴급] 보안 업데이트 필수 안내
MIME-Version: 1.0
Content-Type: multipart/mixed; boundary="boundary123"
...
.
QUIT
```

### 8.3 Roundcube 세션 타임아웃 과다 설정

**취약 구성 위치**: `/etc/roundcube/config.inc.php`

```php
$config['session_lifetime'] = 1440;  // 24시간
$config['ip_check'] = false;          // IP 변경 시에도 세션 유지
$config['use_https'] = false;         // HTTP로 세션 쿠키 전송
```

**공격 활용**: 세션 쿠키를 탈취하면 24시간 동안 다른 IP에서도 피해자의 메일함에 접근할 수 있다. XSS 또는 네트워크 스니핑으로 `roundcube_sessid` 쿠키를 획득하면 된다.

### 8.4 첨부파일 스캐닝/필터링 없음

**취약 구성 위치**: `/etc/postfix/main.cf`, `/etc/roundcube/config.inc.php`

```ini
# Postfix: header_checks / body_checks 미설정
# → .exe, .lnk, .bat, .ps1 등 위험 확장자 필터 없음
# → ClamAV, Amavis 등 안티바이러스 연동 없음
```

```php
// Roundcube: 업로드 MIME 타입 제한 없음
$config['upload_max_filesize'] = '50M';
// → .lnk 파일도 첨부 가능
```

**공격 활용**: 레드팀이 `update_guide.lnk` 악성 LNK 파일을 첨부하여 피싱 메일을 발송할 때 어떤 필터링도 없이 전달된다.

#### 9.2.1 대량 메일 발송 탐지 (피싱 징후)

```bash
# [Postfix] 5분 이내 동일 발신자에서 5건 이상 발송
grep "from=<" /var/log/mail.log | \
  awk -F'from=<' '{print $2}' | cut -d'>' -f1 | \
  sort | uniq -c | sort -rn | head -10

# 시간대별 발송량 집계 (분 단위)
grep "from=<" /var/log/mail.log | \
  awk '{print $1, $2, substr($3,1,5)}' | \
  sort | uniq -c | sort -rn | head -20

# [기준] 정상: 시간당 1~3건 / 이상: 5분 내 5건 이상
```

#### 9.2.2 비정상 로그인 시간 탐지

```bash
# [Dovecot] 비업무 시간대 IMAP 로그인 (22:00~06:00)
grep 'imap-login: Login' /var/log/mail.log | \
  grep -E ' (22:|23:|00:|01:|02:|03:|04:|05:)' | \
  awk '{print $1, $2, $3, $NF}'

# [Roundcube] 비업무 시간 웹 로그인
grep 'POST.*\?_task=login' /var/log/apache2/roundcube-access.log | \
  awk -F'[' '{print $2}' | cut -d']' -f1 | \

```

## 10. 설치/구성 절차 개요

### 10.1 setup.sh 실행 흐름

```bash
#!/bin/bash
# ============================================================
# 웹메일 서버 원클릭 배포 스크립트
# 대상 OS: Debian 12 (Bookworm)
# 실행: sudo bash setup.sh
# ============================================================

set -euo pipefail

# --- 0. 환경변수 로드 ---
source .env

# --- 1. 시스템 업데이트 및 기본 패키지 ---
apt-get update && apt-get upgrade -y
apt-get install -y curl wget gnupg2 apt-transport-https ca-certificates

# --- 2. Postfix 설치 ---
debconf-set-selections <<< "postfix postfix/mailname string ${MAIL_DOMAIN}"
debconf-set-selections <<< "postfix postfix/main_mailer_type string 'Internet Site'"
apt-get install -y postfix

# Postfix 설정 파일 복사
cp conf/postfix/main.cf /etc/postfix/main.cf
cp conf/postfix/master.cf /etc/postfix/master.cf
cp conf/postfix/virtual_mailbox_maps /etc/postfix/virtual_mailbox_maps
postmap /etc/postfix/virtual_mailbox_maps

# --- 3. Dovecot 설치 ---
apt-get install -y dovecot-core dovecot-imapd dovecot-lmtpd dovecot-ldap

# Dovecot 설정 파일 복사
cp conf/dovecot/dovecot.conf /etc/dovecot/dovecot.conf
cp conf/dovecot/conf.d/* /etc/dovecot/conf.d/
cp conf/dovecot/dovecot-ldap.conf.ext /etc/dovecot/dovecot-ldap.conf.ext
chmod 600 /etc/dovecot/dovecot-ldap.conf.ext

# --- 4. 메일 디렉토리 생성 ---
groupadd -g 5000 vmail || true
useradd -g vmail -u 5000 -d /var/mail/vhosts -s /usr/sbin/nologin vmail || true
mkdir -p /var/mail/vhosts/mois.local
chown -R vmail:vmail /var/mail/vhosts

# --- 5. SSL 인증서 생성 ---
bash conf/ssl/generate-cert.sh

# --- 6. Apache + PHP 설치 ---
apt-get install -y apache2 libapache2-mod-php8.2 \
  php8.2 php8.2-fpm php8.2-xml php8.2-mbstring php8.2-intl \
  php8.2-ldap php8.2-sqlite3 php8.2-curl php8.2-zip php8.2-gd

a2enmod proxy_fcgi setenvif rewrite remoteip
a2enconf php8.2-fpm

# --- 7. Roundcube 설치 ---
apt-get install -y roundcube roundcube-core roundcube-plugins roundcube-plugins-extra

# Roundcube 설정 복사
cp conf/roundcube/config.inc.php /etc/roundcube/config.inc.php
chmod 640 /etc/roundcube/config.inc.php
chown root:www-data /etc/roundcube/config.inc.php

# --- 8. Apache VirtualHost 설정 ---
cp conf/apache/roundcube.conf /etc/apache2/sites-available/roundcube.conf
cp conf/apache/roundcube-proxy.conf /etc/apache2/sites-available/roundcube-proxy.conf
a2dissite 000-default.conf
a2ensite roundcube.conf roundcube-proxy.conf

# --- 9. 포트 설정 (8080 추가) ---
if ! grep -q 'Listen 8080' /etc/apache2/ports.conf; then
  echo 'Listen 8080' >> /etc/apache2/ports.conf
fi

# --- 10. 로그 디렉토리 ---
mkdir -p /var/log/roundcube
chown www-data:www-data /var/log/roundcube

# --- 11. 사용자 메일박스 생성 ---
bash seed/seed_mailboxes.sh

# --- 12. 서비스 시작 ---
systemctl restart postfix
systemctl restart dovecot
systemctl restart php8.2-fpm
systemctl restart apache2

systemctl enable postfix dovecot php8.2-fpm apache2

# --- 13. 초기 메일 데이터 삽입 ---
python3 seed/seed_emails.py

# --- 14. 방화벽 (선택) ---
# ufw는 OPNSense가 담당하므로 로컬 방화벽은 비활성
systemctl disable ufw 2>/dev/null || true

echo "============================================"
echo "  웹메일 서버 배포 완료"
echo "  Roundcube: http://${MAIL_HOST}/"
echo "  SMTP: ${MAIL_HOST}:${SMTP_PORT}"
echo "  IMAP: ${MAIL_HOST}:${IMAP_PORT}"
echo "============================================"
```

### 10.2 seed_mailboxes.sh (메일박스 초기화)

```bash
#!/bin/bash
# 사용자별 Maildir 디렉토리 생성

VHOST_DIR="/var/mail/vhosts/mois.local"
USERS=(
  hong.gildong kim.chulsoo lee.younghee park.minsoo choi.jihye
  jung.dongwook yoon.sera kang.jiwon shin.hyunwoo han.minji
  seo.junhyuk admin helpdesk
)

for user in "${USERS[@]}"; do
  mkdir -p "${VHOST_DIR}/${user}/{cur,new,tmp}"
  mkdir -p "${VHOST_DIR}/${user}/.Sent/{cur,new,tmp}"
  mkdir -p "${VHOST_DIR}/${user}/.Drafts/{cur,new,tmp}"
  mkdir -p "${VHOST_DIR}/${user}/.Trash/{cur,new,tmp}"
  mkdir -p "${VHOST_DIR}/${user}/.Junk/{cur,new,tmp}"
done

chown -R vmail:vmail "${VHOST_DIR}"
echo "메일박스 생성 완료: ${#USERS[@]}개 계정"
```

### 11.4 초기 메일 삽입 스크립트 (`seed/seed_emails.py` 개요)

```python
#!/usr/bin/env python3
"""
초기 메일 데이터 삽입 스크립트
seed/emails/ 디렉토리의 .eml 파일을 각 사용자 Maildir에 삽입
"""

import os
import imaplib
import email
import glob

IMAP_HOST = os.getenv('MAIL_HOST', '192.168.100.11')
IMAP_PORT = int(os.getenv('IMAP_PORT', '143'))
MAIL_DOMAIN = 'mois.local'
EML_DIR = os.path.join(os.path.dirname(__file__), 'emails')

# 각 사용자에게 배달할 메일 매핑
DELIVERY_MAP = {
    'hong.gildong':  ['02', '03', '05', '07'],
    'kim.chulsoo':   ['02', '03', '05'],
    'lee.younghee':  ['03', '05', '09'],
    'park.minsoo':   ['02', '03', '04', '05'],
    'choi.jihye':    ['03', '05'],
    'jung.dongwook': ['03', '05'],
    'yoon.sera':     ['03', '05', '06'],
    'kang.jiwon':    ['03', '05', '07'],
    'shin.hyunwoo':  ['03', '05', '06'],
    'han.minji':     ['06', '08'],
    'seo.junhyuk':   ['03', '05', '10'],
    'helpdesk':      ['04', '08', '10'],
}

# 전체 직원에게 배달할 메일
ALL_STAFF_MAILS = ['01', '03', '05', '06']

def deliver_mail(user, password, eml_path):
    """IMAP APPEND로 메일 삽입"""
    with open(eml_path, 'rb') as f:
        msg_data = f.read()

    conn = imaplib.IMAP4(IMAP_HOST, IMAP_PORT)
    conn.login(f'{user}@{MAIL_DOMAIN}', password)
    conn.append('INBOX', None, None, msg_data)
    conn.logout()

if __name__ == '__main__':
    # 각 사용자별 메일 삽입
    for user, mail_ids in DELIVERY_MAP.items():
        for mid in mail_ids:
            eml_files = glob.glob(f'{EML_DIR}/{mid}_*.eml')
            for eml in eml_files:
                print(f'  Delivering {eml} → {user}@{MAIL_DOMAIN}')
                # 실제 배포 시 deliver_mail() 호출
    print('초기 메일 데이터 삽입 완료')
```
