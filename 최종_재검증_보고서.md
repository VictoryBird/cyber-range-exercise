# 🤖 AI 개발 위임 자산 — 상세 검증

## AI 개발 가능 자산 목록 (12개)

### [공공기관 DMZ-1] 외부 포털 서버
- **스택:** React + Nginx + FastAPI
- **AI 개발 범위:** FastAPI 백엔드 전체, React 프론트, 취약점 삽입(인증 미적용 엔드포인트, SQLi)
- **직접 작업:** Nginx 설정, 서버 배포
- **공격 흐름 연결:** 정찰 → API 취약점 발견 → 공공기관 DMZ 초기 접근
- **검증:** ✅ AI가 FastAPI 라우터에 `@app.get("/api/admin/users")`(인증 없음) 삽입하는 것은 매우 용이. 공격 플레이북 STEP 1-1과 직접 매칭
- **주의:** 외부 포털이 DMZ 침투의 보조 경로이고, **민원접수서버(3번)가 메인 침투 경로**임을 AI에게 명시할 것

---

### [공공기관 DMZ-3] 민원 접수 서버 ⭐ 핵심
- **스택:** React + Nginx + FastAPI + MinIO
- **AI 개발 범위:** 민원 제출 폼, 파일 업로드 API, 업로드 검증 로직(의도적 취약), MinIO 연동
- **취약점 삽입 포인트:**
  - 확장자만 검사(콘텐츠 미검증): `if file.filename.endswith('.pdf'): allow()`
  - MinIO presigned URL 권한 오설정 → 타인 파일 접근 가능
  - 처리 상태 조회 API에 IDOR 취약점
- **공격 흐름 연결:** **공격 시나리오의 핵심 초기 침투 벡터** → 민원 처리 서버 자동 실행
- **검증:** ✅ 플레이북 STEP 1-3과 완전히 매칭. 업로드 후 파이프라인 자동 실행까지 이어지는 흐름이 가장 자연스러운 구간

---

### [공공기관 INT-4] 내부 업무 포털
- **스택:** Django 또는 Spring Boot + PostgreSQL
- **AI 개발 범위:** 결재/공지/업무요청 CRUD, 사용자 인증, 취약점 삽입
- **취약점 삽입 포인트:**
  - IDOR: `/api/user/{id}/profile` → 타 사용자 조회 가능
  - 인가 미적용 관리 API: `/api/admin/export-users`
  - 세션 고정 취약점
- **공격 흐름 연결:** 메일 게이트 우회 또는 민원처리 서버 침해 후 → 업무 포털 계정 탈취 → 웹메일 접근
- **검증:** ✅ 플레이북 STEP 2-3과 매칭. **업무포털과 웹메일 계정을 동일하게 설정**하는 것이 핵심 — AI에게 이 설정을 명시해야 함

---

### [공공기관 INT-6] 민원 처리 서버 ⭐ 핵심
- **스택:** Python Worker + Redis/RabbitMQ + 파일변환도구
- **AI 개발 범위:** 파일 수신 워커, 변환 파이프라인, 큐 연동 로직
- **취약점 삽입 포인트:**
  - 파일 변환 도구 취약 버전 사용 (LibreOffice headless, Pillow)
  - 파일 처리 전 콘텐츠 검증 없음
  - 서비스 계정 과권한 (`application.properties`에 DB SUPERUSER 계정)
- **공격 흐름 연결:** 민원접수서버 업로드 → 이 서버 파이프라인에서 코드 실행 → DB 탈취 → INT 확산
- **검증:** ✅ 시나리오 전체에서 가장 중요한 전환점. 블루팀 탐지 SCENARIO 02와 완전 매칭
- **주의:** `application.properties`에 DB 크리덴셜 하드코딩 설정이 8번 DB서버 공격과 직접 연결되므로 반드시 포함

### [공공기관 INT-8] DB 서버 (취약 설정 스크립트)

- **공격 흐름 연결:** 민원처리서버 설정파일 탈취 → 이 DB에 직접 접속 → COPY TO 덤프
- **검증:** ✅ 플레이북 STEP 2-1과 정확히 매칭. 구현 난이도 낮음

  ```nginx

  }
  ```
- **검증:** ✅ 간단하지만 공격 흐름에서 웹메일 직접 접근 경로로 중요

### [공공기관 INT-7] AI Assistant (llama3 + OpenWebUI)
- **구축 내용:** llama3 모델 다운로드, OpenWebUI 설치, Nextcloud RAG 연동
- **난이도:** 중 — GPU 스펙에 따라 llama3 응답 속도 차이
- **핵심 설정:**

- **검증:** ✅ 구현 가능. **GPU 없으면 llama3 대신 llama3.2:1b 또는 gemma:2b로 경량화 가능**

# ⚠️ 발견된 이슈 및 조치 사항

## 이슈 1 — AI Assistant 접근 경로 선행 조건
**문제:** 공격자가 AI Assistant(7번)에 접근하려면 업무 포털 계정이 먼저 필요한데, 업무 포털 침해가 민원처리서버 경유 또는 메일 게이트 우회 둘 중 하나로 이루어져야 함

**조치:** 두 경로 모두 열어두되, **민원처리서버 → 업무포털 계정 탈취 → AI Assistant** 순서를 메인 경로로 설정. 레드팀 플레이북에 이미 반영됨 ✅

---

## 이슈 2 — 공공기관 INT → Industrial DMZ 방화벽 통과
**문제:** OPNSense-4가 Industrial DMZ 경계를 막고 있는데, INT에서 Historian API 접근이 가능한 설정이 필요

**조치:** OPNSense-4에 `INT(10.10.0.0/24) → Historian API(10.20.0.10:8000)` 허용 규칙 명시적으로 추가. 역방향(Industrial DMZ→INT)은 차단 유지

---

## 이슈 3 — 군 VPN → C4I 직접 접근 불가
**문제:** VPN 인증 성공 시 패치관리서버 서브넷만 허용 → C4I는 어떻게 접근?

**조치:** 군 패치관리서버 → 업무용 PC 감염 → PC에서 C4I 접근 경로가 이미 시나리오에 존재. VPN이 패치관리서버 전용인 것은 맞고, C4I는 군 INT 확산 후 별도 경로로 접근. **흐름에 문제 없음** ✅

---

## 이슈 4 — 상황요약 AI(C4I-15) LLaMA 실행 환경
**문제:** LLaMA 로컬 실행에 GPU 또는 고사양 CPU 필요

**조치:** `llama3.2:1b` 또는 `gemma2:2b` 경량 모델로 대체 가능. 요약 기능 자체는 단순하므로 경량 모델로도 충분. AI 개발 위임 시 이 제약 명시 필요

---

## 이슈 5 — 블루팀 탐지 시나리오와 레드팀 철수 간 충돌
**문제:** 레드팀이 로그를 완전히 삭제하면 블루팀 분석 불가

**조치:** 레드팀 플레이북에 "일부 로그만 삭제" 지침 포함. Application 로그는 남기고 Security 로그 일부만 삭제하도록 명시. 블루팀은 Event ID 1102(로그 삭제 이벤트) 자체가 공격 흔적임을 파악해야 함 ✅
