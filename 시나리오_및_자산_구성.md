# 🌐 네트워크 인프라

## OPNSense 방화벽 구성 (존별 1개, 총 7개)

```
[인터넷/외부]
      │
  OPNSense-1 (공공기관 DMZ 경계)
      │
  공공기관 DMZ
      │
  OPNSense-2 (공공기관 DMZ ↔ INT 경계)
      │
  공공기관 INT ──── OPNSense-4 (Industrial DMZ 경계)
      │                    │
  OPNSense-3          Industrial DMZ
  (군 DMZ 경계)            │
      │               OPNSense-5 (OT 경계)
  군 DMZ(EXT)              │
      │                   OT
  OPNSense-6 (군 INT 경계)
      │
  군 INT
      │
  OPNSense-7 (C4I 경계)
      │
    C4I
```

- VMware 환경에서 운영 (OPNSense VM 1개당 메모리 1~2GB)
- 각 존 경계마다 방화벽 로그 분리 → 블루팀 분석 훈련 가치 극대화

## Industrial DMZ 통신 방향 설계

```
공공기관 INT → (허용) → Industrial DMZ (Historian API)
Industrial DMZ → (허용) → OT (SCADA 쿼리)
OT → (허용) → Industrial DMZ (데이터 쓰기)
Industrial DMZ → (차단) → 공공기관 INT (역방향 차단)
```

- 역방향은 OPNSense-4에서 차단
- 공격 시나리오: Historian 침해 후 OPNSense 규칙 변경 또는 피벗 포인트로 OT 접근

## 군 패치관리서버 서브넷 구성

```
군 DMZ (원격접속 게이트웨이)
      │  VPN 인증 성공 시
      ▼
  패치관리서버 전용 서브넷만 라우팅 허용 (OPNSense-6 규칙)
      │
  패치관리서버 (군 INT)
```

## 훈련 운영 방식

- **레드팀 선공 / 블루팀 사후 로그 분석** 방식
- 로그 보존 기간: 훈련 전체 기간보다 길게 설정 필수
- 중앙 로그 수집: Wazuh 또는 Graylog 구성 권장 (별도 자산 추가 검토)
- VMware 스냅샷으로 훈련 후 자산 원복

## 공공기관 → 군 전환 경로

- **공공기관 AI Assistant(7번)가 핵심 피벗 포인트**
- 프롬프트 인젝션으로 Nextcloud 내 군 연계 문서 3종에서 정보 수집
- 도출 정보로 군 VPN 초기 계정 크리덴셜 획득 → 군 DMZ 침투
