## 1. 개요

### 1.1 자산 목적

C13 작전 상황도 서버는 COP(Common Operational Picture) 웹 애플리케이션을 호스팅한다. 전장 상황을 실시간 지도에 시각화하여 작전 지휘관과 참모진이 현재 전장 상태를 한눈에 파악할 수 있도록 한다.

주요 기능:
- **전체화면 작전 지도**: OpenLayers 7 기반 전자 지도 (한반도 중심)
- **부대 마커**: 아군(청색), 적군(적색), 중립(녹색) 색상 구분 부대 마커
- **작전 요소**: 방어선, 작전 구역, 이동 경로 시각화
- **이벤트 타임라인**: 시간순 이벤트 목록 표시
- **관리자 패널**: 지도 객체 CRUD (추가/수정/삭제)
- **검색/필터**: 부대 유형별, 시간별 필터링

### 1.2 훈련에서의 역할

이 자산은 STEP 5-3에서 **COP 지도 직접 변조** 대상이다. JSP 페이지의 SQL Injection 취약점을 통해 데이터베이스의 `map_objects` 테이블을 직접 조작하여 허위 적군 마커를 삽입하고 아군 마커를 삭제할 수 있다. 또한 Stored XSS를 통해 작전 요원의 브라우저에서 악성 스크립트를 실행할 수 있다.

## 4. 데이터 모델

### 4.1 PostgreSQL 스키마

```sql
-- ============================================================
-- C13 작전 상황도 서버 — PostgreSQL 스키마
-- 데이터베이스: cop_db
-- ============================================================

-- 데이터베이스 및 사용자 생성
CREATE USER cop_user WITH PASSWORD 'C0p!Map#2024';
CREATE DATABASE cop_db OWNER cop_user;

\c cop_db

-- ============================================================
-- map_objects: 지도 위 표시 객체 (부대 마커, 방어선, 구역)
-- ============================================================
CREATE TABLE map_objects (
    id              SERIAL PRIMARY KEY,
    obj_type        VARCHAR(20) NOT NULL,       -- 'unit', 'line', 'area'
    sub_type        VARCHAR(30),                -- 'infantry', 'armor', 'artillery', 'defense_line', 'op_area'
    affiliation     VARCHAR(10) NOT NULL,       -- 'friendly', 'enemy', 'neutral'
    label           VARCHAR(200) NOT NULL,      -- ★ Stored XSS 가능 (미검증)
    position_lat    DOUBLE PRECISION,
    position_lng    DOUBLE PRECISION,
    positions_json  TEXT,                        -- 다중 좌표 (라인/영역용, JSON 배열)
    color           VARCHAR(7) DEFAULT '#0066CC',
    icon            VARCHAR(50) DEFAULT 'circle',
    metadata        JSONB DEFAULT '{}',
    status          VARCHAR(20) DEFAULT 'active', -- 'active', 'inactive', 'destroyed'
    created_at      TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================================
-- events: 작전 이벤트 (타임라인 표시용)
-- ============================================================
CREATE TABLE events (
    id              SERIAL PRIMARY KEY,
    event_type      VARCHAR(30) NOT NULL,       -- 'friendly_move', 'enemy_advance', 'artillery_fire', 'air_support'
    description     TEXT,
    unit_name       VARCHAR(100),
    location_lat    DOUBLE PRECISION,
    location_lng    DOUBLE PRECISION,
    event_time      TIMESTAMP DEFAULT NOW(),
    source          VARCHAR(20) DEFAULT 'relay', -- 'relay', 'manual', 'sensor'
    priority        VARCHAR(10) DEFAULT 'medium', -- 'low', 'medium', 'high', 'critical'
    created_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================================
-- audit_log: 변경 이력 (있으나 모니터링 안 함 — 설계적 취약)
-- ============================================================
CREATE TABLE audit_log (
    id              SERIAL PRIMARY KEY,
    table_name      VARCHAR(50),
    record_id       INTEGER,
    action          VARCHAR(10),                -- 'INSERT', 'UPDATE', 'DELETE'
    old_data        JSONB,
    new_data        JSONB,
    changed_by      VARCHAR(50),
    changed_at      TIMESTAMP DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_map_objects_affiliation ON map_objects(affiliation);
CREATE INDEX idx_map_objects_type ON map_objects(obj_type);
CREATE INDEX idx_events_type ON events(event_type);
CREATE INDEX idx_events_time ON events(event_time);
```
