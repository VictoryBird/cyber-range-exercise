## 1. 개요

### 1.1 자산 목적

내부 업무 포털은 발도리아 국방부(MND) 군 내부 직원이 일상 업무를 수행하는 인트라넷 웹 애플리케이션이다. 전자정부 프레임워크(eGovframework)와 Spring Boot를 기반으로 구축되며, Active Directory(corp.mnd.local) 통합 인증을 지원한다.

주요 기능:

- **로그인/인증**: AD(LDAP) 연동 SSO
- **대시보드**: 부서별 업무 현황, 공지사항 요약
- **공지사항**: 군 내부 공지 등록/열람
- **전자 결재**: 기안 작성, 결재선 설정, 승인/반려
- **문서 관리**: 내부 문서 등록/검색/열람
- **사용자/조직 관리**: 군 인원 프로필, 부서 정보, 조직도

### 1.2 훈련에서의 역할

이 자산은 군 INT 영역의 기반 인트라넷으로, 소스코드 취약점(인증/인가 미흡, SQL Injection, 파일 업로드)이 의도적으로 삽입되어 공격자에게 추가 공격 표면을 제공한다. 다만 eGovframework 특성상 구현 난이도가 높으므로 **마지막에 구현**하며, 설계는 상위 수준으로 유지한다.

### 1.3 핵심 공격 벡터 요약

| 취약점 | 유형 | 공격 효과 |
|--------|------|-----------|
| 인증 우회 | /api/admin/* 인증 미적용 | 관리자 기능 무단 접근 |
| SQL Injection | /api/search?keyword= | DB 전체 덤프 (users 테이블) |

## 6. 취약점 상세 (설계)

### 6.1 SQL Injection — /api/search

```java
// ⚠ 취약: raw query에 사용자 입력 직접 삽입
@GetMapping("/api/search")
public ResponseEntity<?> search(@RequestParam String keyword) {
    String sql = "SELECT id, title, content FROM notices WHERE title LIKE '%" + keyword + "%' " +
                 "UNION SELECT id, username, email FROM users WHERE username LIKE '%" + keyword + "%'";
    List<Map<String, Object>> results = jdbcTemplate.queryForList(sql);
    return ResponseEntity.ok(results);
}
```

### 6.2 파일 업로드 — /api/documents/upload

```java
// ⚠ 취약: 확장자 검증 없이 업로드 허용
@PostMapping("/api/documents/upload")
public ResponseEntity<?> uploadDocument(@RequestParam("file") MultipartFile file) {
    String filename = file.getOriginalFilename();
    // 확장자 필터링 없음 — .jsp, .war 업로드 가능
    Path target = Paths.get("/opt/mil-portal/uploads/" + filename);
    Files.copy(file.getInputStream(), target);
    return ResponseEntity.ok(Map.of("path", "/uploads/" + filename));
}
```

### 6.3 관리자 API 무인증

```java
// ⚠ 취약: 인증/인가 어노테이션 누락
@GetMapping("/api/admin/export-users")
public ResponseEntity<?> exportUsers() {
    List<User> users = userRepository.findAll();
    return ResponseEntity.ok(users); // 전 직원 정보 노출
}

@PostMapping("/api/admin/create-user")
public ResponseEntity<?> createUser(@RequestBody UserCreateDto dto) {
    // 인증 없이 사용자 생성 가능 → 백도어 계정 생성
    User user = new User(dto);
    userRepository.save(user);
    return ResponseEntity.ok(Map.of("status", "created"));
}
```

## 8. 설치 절차 (개요)

> **참고: eGovframework 부분은 "마지막에 구현" — 아래는 상위 수준 설계이다.**

```bash
#!/bin/bash
# I6 내부 업무 포털 설치 스크립트 (개요)
set -euo pipefail

# 1. Rocky Linux 9 기본 패키지
dnf install -y java-17-openjdk java-17-openjdk-devel nginx postgresql-server

# 2. PostgreSQL 초기화
postgresql-setup --initdb
systemctl enable --now postgresql
sudo -u postgres psql -c "CREATE DATABASE mnd_intranet;"
sudo -u postgres psql -c "CREATE USER portal_user WITH PASSWORD 'PortalDB2026!';"
sudo -u postgres psql -c "GRANT ALL ON DATABASE mnd_intranet TO portal_user;"
sudo -u postgres psql -d mnd_intranet -f /opt/mil-portal/sql/schema.sql
sudo -u postgres psql -d mnd_intranet -f /opt/mil-portal/sql/seed.sql

# 3. Spring Boot JAR 배포
cp mil-portal/build/libs/mil-portal.jar /opt/mil-portal/
cp systemd/mil-portal.service /etc/systemd/system/
systemctl enable --now mil-portal

# 4. Nginx 설정
cp conf/nginx/mil-portal.conf /etc/nginx/conf.d/
systemctl enable --now nginx

echo "[완료] I6 내부 업무 포털 배포 완료"
```
